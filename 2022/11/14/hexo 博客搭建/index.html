hexo 博客搭建

发表于2022-11-14|更新于2022-11-15|杂项
|字数总计:1.1k|阅读时长:3分钟|阅读量:|评论数:

一、前置要求

  • node.js v12 版本以上
  • git

安装 hexo

  • 全局安装 npm install -g hexo-cli
  • 局部安装 npm i hexo 将 hexo 所在 node_modules 配置环境变量之中即可使用 hexo 命令

检验 终端输入 hexo 命令，有如下提示即说明安装成功

二、建站

  1. 在指定路径下使用hexo init <folder name>创建网站根目录
  2. cd <folder name>
  3. npm i 安装依赖

完成后的目录大概如下，(本人用的是 pnpm，稍有不同)

下面重点介绍一下每个文件夹有什么用。

  1. .**github**应该是表明 hexo 的版本信息相关的，内容如下

  1. **node_modules **熟悉 node 的都知道，存放的是依赖文件，npm i 下载的都在这里
  2. **scaffolds**** **模板文件夹，新建文章时，hexo 会根据 scaffold 来建立文件。

  1. **source**文件夹，资源目录，除**_posts **文件夹外，开头命名为**_**的文件或文件夹会被忽略。md 文件和 html 文件会被解析到**public**(通过 hexo g 生成)文件夹

  1. **themes**主题文件夹，hexo 根据主题生成静态页面

  1. **_config.yml** 配置文件，存放网站的配置信息，详细可见官网
  2. **package.json** 存放依赖关系的文件

使继续在该目录下使用 hexo 命令 hexo generate或缩写hexo g生成静态页面，此时目录中多了一个 public 文件夹。
继续运行**hexo s**命令，启动服务，可以查看建立起的网站。

使用hexo new "title"来生成博客文章，默认是 markdown 格式。生成的文件在 source/_posts 目录下

四、配置文件

_config.yml 文件，有网站配置文件和主题配置文件
主题配置文件在 themes 的对应主题文件夹中，网站配置文件在根目录。
各配置项官网有，且很详细。

五、常用命令

  • hexo init 新建一个网站
  • hexo generate生成静态文件
    • -d，--deploy 文件生成后立即部署
    • -w，--watch 监视文件变动，有变动就会重新 generate
  • hexo publish draftname 发布草稿
  • hexo server 启动服务器
    • -p，--port 重设端口
    • -s，--static只使用静态文件
    • -l，--log生成日志
  • hexo deploy部署网站
  • hexo render file 渲染单个文件，与 generate 的区别是 generate 会生成所有文件。
  • hexo clean清楚缓存或者已生成的静态文件
  • hexo list <type> 列出网站信息

常用选项

  • --safe 不使用任何插件和脚本

六、部署到 github

使用的是官网提供的一个一键部署方案。步骤如下：

  1. 在网站配置文件中找到如下配置项并进行修改，注意格式语法不要错，详见官网
  1. 安装npm install hexo-deployer-git --save
  2. 执行hexo clean hexo g hexo d
  3. 修改github pages 的默认分支为你推送的分支

  1. 访问网站即可看到发布的博客了

  1. 此时，新建完文章后，就可以通过hexo g和hexo d发布到网站了
