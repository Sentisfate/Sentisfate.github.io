center 不生效

view的默认投影是**ESPG:3857**(伪墨卡托投影，也叫球体墨卡托投影，**Web Mercator**)，而我们常使用的是**EPSG:4326** 也就是WGS84。我们一般使用web墨卡托显示数据，用WGS84存储数据。

**EPSG:3857**是一种投影坐标系，它的数据一般是这种的。**[12914838.35,4814529.9]**，单位是m，看上去相对数值较大。不利于存储，比较占内存。 经度[-180°, 180°]，纬度[-85°, 85°]对应x[-20037508.3427892, 20037508.3427892]，范围y同��是[-20037508.3427892, 20037508.3427892]，单位为米。 **EPSG:4326** 是一种地理坐标系，它的数据一般是这种的。**[22.37，114.05]**单位是经纬度。利于存储，可读性高，但是页面变形大，两极区域变形极大

需要注意的是，对于墨卡托投影来说，越到高纬度，大小扭曲越严重，到两极会被放到无限大，所以，墨卡托投影无法显示极地地区。另外，伪墨卡托投影还切掉了南北85.051129°纬度以上的地区，以保证整个投影是正方形的。因为墨卡托投影等正形性的特点，在不同层级的图层上物体的形状保持不变，一个正方形可以不断被划分为更多更小的正方形以显示更清晰的细节。 :::info 不生效的原因:其实是生效了，但是数值太小，肉眼看不出来。解决方案:

  1. 将投影设置为 projection:‘EPSG:4326’
  2. 将WGS84坐标转换为Web墨卡托
    1. 使用transform转换坐标。**center: ol.proj.transform([114, 30], 'EPSG:4326', 'EPSG:3857'),**
    2. 使用fromLonLat 转换经纬度为投影坐标,center:** ol.proj.fromLonLat([114, 30])**,
    3. 调用 **useGeographic()**** ** :::

Use geographic coordinates (WGS-84 datum) in API methods. This includes all API methods except for those interacting with tile grids.
