什么是WebGL

WebGL 使得在支持 HTML 的 canvas 标签的浏览器中，不需要安装任何插件，便可以使用基于 OpenGL ES 2.0 的 API 在 canvas 中进行 2D 和 3D 渲染。WebGL 程序包括用 JavaScript 写的控制代码，以及在图形处理单���（GPU, Graphics Processing Unit）中执行的着色代码（GLSL，注：GLSL 为 OpenGL 着色语言）。WebGL 元素可以和其他 HTML 元素混合使用，并且可以和网页其他部分或者网页背景结合起来。

webgl的本质是javascript操作opengl的一个接口，是一个光栅化引擎。

基础概念

Webgl 本质上是一个光栅化引擎，它可以根据你提供的代码绘制点，线，三角形。至于更复杂的图形，则是根据点线三角形进行组合而成。

光栅化Rsterization引擎，

WebGL在GPU中运行，GPU需要的方法或者叫着色程序由一个顶点着色器和一个片段着色器组成，而着色器使用一种类似C的强类型语言GLSL。顶点着色器用于计算顶点位置，片段着色器计算当前图元的像素的颜色值。WebGL API在要绘制一个对象时，需要设置这些着色器的一系列状态值，在调用gl.drawArrays或者gl.drawElements运行一个着色方法对。顶点和片段着色器需要的数据都需要发送到GPU，而着色器获取数据共有4种方法：

  1. 属性（Attributes）和缓冲（Buffer）

缓冲是发送到GPU的一些二进制数据序列，包含位置，法向量，纹理坐标，顶点颜色等，在js中通常用TypedArray来存储并传递数据。属性用来表明怎么从缓冲中获取数据并提供给着色器，例如当用3个32位浮点型数据存储位置值，需要告诉着色器从哪个缓冲区，从第几位（偏移多少位）开始，什么类型，读几个数据。顶点着色器运行的次数是一个指定的确切数字，每次运行属性会从指定的缓冲区按照指定规则读取数据，即确定属性后顶点着色器运行是可以推定的。

  1. 全局变量 Uniforms

全局变量在着色器运行前赋值，在运行过程中全局有效

  1. 纹理 Textures

纹理是一个数据序列，可以在着色程序运行中随意读取其中的数据，多数存放的是图像数据，纹理仅仅是数据序列，可以存放除了颜色数据以外的其它数据。

  1. 可变量 Varyings

可变量是一种顶点着色器给片段着色器传值的方式，依照渲染图元是点，线，还是三角形，顶点着色器中设置的可变量会在片段着色器运行中获取不同的插值。

WebGL只关心两件事，裁剪空间中的坐标值和颜色值。顶点着色器提供裁剪空间中的坐标值，片段着色器提供颜色值。一个顶点着色器可以像下面这样：

// 属性值，
attribute vec4 a_position;
void main(){
  // 变换后的顶点位置，用于后续操作 
	gl_Position=a_position;
}

片段着色器：

//片段着色器没有默认精度，所以需要设置一个精度
//mediump是一个不错的默认值，代表"mediump precision" 中等精度
precision mediump float;

void main(){
  //gl_FragColor是片段着色器内置的一个用于输出颜色的内置变量
  gl_FragColor=vec4(1,0,0.5,1);
}

总结一下webgl的大致绘制流程：

其中主体步骤大致分为5部分，中间细节可能有些差异。

  1. 创建webgl渲染上下文
  2. 创建着色器
  3. 创建着色程序
  4. 为着色程序提供数据
  5. 进行渲染

仔细研究下过程中会遇到的一些api

  • getContext(contextType,contextAttributes) 创建webgl上下文
    • contextType:
      • '2d'创建一个CanvasRenderingContext2D上下文对象
      • 'webgl'创建 WebGLRenderingContextwebgl1(OpenGL ES 2.0)
      • 'webgl2'创建WebGL2RenderingContextwebgl2(openGL ES 3.0)
      • 'webgpu'创建一个GPUCanvasContext上下文对象，涉及到WebGPU API
      • 'bitmaprenderer' 创建一个ImageBitmapRenderingContext上下文，参看ImageBitmap
    • contextAttributes optional object
interface ContextAttributes{
  alpha:boolean;//表示是否有透明度这个颜色通道   默认 true
  depth:boolean;//是否开启深度缓冲区			false

  antialias:boolean;//是否启用抗锯齿，		true

  preserveDrawingBuffer:boolean;//是否保留绘图缓冲区    false
  failIfMajorPerformanceCaveat:boolean;//是否在性能差的时候避免创建上下文  false
}
  • createShader(type)返回一个指定类型的着色器对象
    • type:gl.VERTEX_SHADER|gl.FRAGMENT_SHADER,顶点着色器或者片段着色器
  • shaderSource(shader,source)
    • shader:着色器对象
    • source:glsl源代码字符串
  • compileShader(shader)编译着色器源码
  • getShaderParameter(shader,pname)返回着色器参数
    • shader着色器对象
    • pname要获取参数的一个常量
      • gl.COMPILE_STATUS是否成功编译
      • gl.SHADER_TYPE着色器类型
      • gl.DELETE_STATUS是否已成功删除
      • gl.INFO_LOG_LENGTH信息日志的的长度，一个整数
      • gl.SHADER_SOURCE_LENGTH返回一个整数，表示着色器源码的长度
  • createProgram()初始化一个webglprogram对象
  • attachShader(program,shader)将顶点或者片段着色器附加到着色程序上
  • linkProgram(program)链接一个着色程序
  • getProgramParameter(program,pname)返回要查询的程序的属性状态
    • program着色程序对象
    • pname要查询的参数常量
      • gl.DELETE_STATUS是否已经删除
      • gl.LINK_STATUS是否成功链接
      • gl.VALIDATE_STATUS程序对象是否已经通过验证
      • gl.ATTACHED_SHADERS程序附加的着色器数量
      • gl.ACTIVE_ATTRIBUTES 程序对象中活动统一变量数量。
      • gl.ACTIVE_UNIFORMS活动的全局变量数量。
  • getAttribLocation(program,name)返回着色程序中的属性变量的位置索引
  • createBuffer()创建一个顶点或颜色缓冲
  • bindBuffer(target,buffer)
    • target表示缓冲区绑定目标的常量
      • gl.ARRAY_BUFFER包含顶点属性的缓冲区，要包含顶点属性，如顶点坐标，纹理，顶点颜色
      • gl.ELEMENT_ARRAY_BUFFER包含顶点索引数据的缓冲区

gl.COPY_READ_BUFFER

gl.PIXEL_PACK_BUFFER
gl.PIXEL_UNPACK_BUFFER
  • buffer顶点或者颜色缓冲区对象
  • bufferData(target,srcData,usage)将数据通过绑定点写入缓冲对象
    • target绑定点
      • gl.ARRAY_BUFFER包含顶点属性的缓冲区，要包含顶点属性，如顶点坐标，纹理，顶点颜色
      • gl.ELEMENT_ARRAY_BUFFER包含顶点索引数据的缓冲区
    • srcDataArrayBuffer或者TypedArray 或者DataView，其中的数据将会被拷贝进绑定点绑定的缓冲中
    • usage表示数据使用方式的常量，意在帮助实现更好的性能，并不代表一定。
      • gl.STATIC_DRAW表示数据是静态的，不会经常改变，因此程序只会向缓冲区写入一次数据，但要绘制很多次。
      • gl.DYNAMIC_DRAW数据会周期性更改，多次绘制
      • gl.STREAM_DRAW缓冲区会被频繁更改
  • viewport(x,y,width,height)用于设置视口的位置和大小，视口的默认位置为0，大小为画布尺寸，但当画布尺寸发生变化时，视口的大小并不会改变，因此某些情况下需要动态设置。
    • x：视口左下角x坐标
    • y：视口左下角y坐标
    • width：视口的宽度
    • height：视口的高度
  • clearColor(r,g,b,a)指定用于清除颜色缓冲区时用的值，颜色分量均为0-1
  • clear(mask)将缓冲区清除为预设值
    • mask要清除的缓冲区的标识符
      • gl.COLOR_BUFFER_BIT清除颜色缓冲区，清除画布的颜色
      • gl.DEPTH_BUFFER_BIT清除深度缓冲区，清除画布上的深度信息
      • gl.STENCIL_BUFFER_BIT清除模板缓冲区，即清除画布上的模板信息
  • useProgram(program)告诉webgl执行哪个着色程序
  • enableVertexAttribArray(index)启用索引对应的属性
  • vertexAttribPointer(index,size,type,normalized,stride,offset)配置顶点属性指针，告诉webgl如何解析着色器的某一属性
    • index某一属性的位置指针
    • size每个顶点属性组成的个数，只能为1，2，3，4
    • type指定顶点每一部分数据类型
      • gl.BYTE
      • gl.SHORT
      • gl.UNSIGNED_BYTE
      • gl.UNSIGNED_SHORT
      • gl.FLOAT
      • gl.HALF_FLOAT
      • gl.INT
      • gl.UNSIGNED_INT
      • gl.INT_2_10_10_10_REV
      • gl.UNSIGNED_INT_2_10_10_10_REV
    • normalized是否将数据进行归一化处理
      • 对于gl.BYTE和gl.SHORT,归一化为[-1，1]
      • 对于gl.UNSIGNED_BYTE和gl.UNSIGNED_SHORT将值归一化到[0,1]
      • 浮点类型无效
    • stride：每一属性之间间隔的内存空间，为0表示紧凑布局
    • offset：从缓冲区起始位置偏移几位
  • drawArrays(mode,first,count)根据给定的顶点数据绘制图形
    • mode 绘制的方式
      • gl.POINTS点
      • gl.LINES线段，每两个顶点画一条线
      • gl.LINE_STRIP绘制一系列连接的线段，后一个顶点连接前一个顶点，第一个顶点不会和最后一个顶点连接
      • gl.TRIANGLES单独的三角形，每三个顶点绘制一个
      • gl.TRIANGLES_STRIP三角带，每个顶点与前两个顶点相连
      • gl.TRIANGLE_FAN，扇形状三角形，每个顶点与第一个顶点和前一个顶点相连形成三角形
    • first表示从顶点数组的第几个索引开始绘制
    • count表示要绘制的顶点的个数

      const a_colorLocation = gl.getUniformLocation(program, "a_color");
      const positions = [60, 60, 400, 100, 100, 200];
      const buffer = gl.createBuffer();
      gl.bindBuffer(gl.ARRAY_BUFFER, buffer);
      gl.bufferData(
        gl.ARRAY_BUFFER,
        new Float32Array(positions),
        gl.STATIC_DRAW
      );
      // 重置裁剪空间
      gl.viewport(0, 0, gl.canvas.width, gl.canvas.height);
      gl.clearColor(0, 0, 0, 0.1);
      gl.clear(gl.COLOR_BUFFER_BIT);
      gl.useProgram(program);
      //
      gl.enableVertexAttribArray(a_position);
      gl.bindBuffer(gl.ARRAY_BUFFER, buffer);
      gl.uniform2f(
        resolutionUniformLocation,
        gl.canvas.width,
        gl.canvas.height
      );
      gl.uniform2f(centerLocation, center.x, center.y);
      gl.uniform4f(a_colorLocation, 1, 0, 0.5, 0.2);
      // 配置属性指针
      gl.vertexAttribPointer(a_position, 2, gl.FLOAT, false, 0, 0);

      // 开始渲染
      gl.drawArrays(gl.POINTS, 0, 3);

        gl.compileShader(shader); //编译着色器

      float tx=-2.0*x/w;
