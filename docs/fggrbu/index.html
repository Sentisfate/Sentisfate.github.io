git基本理论

git 本地有三个工作区，工作目录(Working Directory)，暂存区(Stage/index)，资源库(Repository)，加上远程git仓库(Remote Directory)可以分为四个工作区。

  1. 工作目录：平时放代码的地方
  2. index/Stage 暂存区，临时存放改动
  3. Repository，本地仓库
  4. Remote Direcotry，远程仓库

工作流程

git的工作流程一般如下：

  1. 在工作目录中添加，修改文件
  2. 将需要版本控制的文件放图暂存区
  3. 将暂存区文件提交到git仓库
  4. 本地仓库推送给远程仓库

常用命令

本地仓库搭建 git init 多了一个.git文件夹克隆远程仓库git clone <url>

git文件操作

git文件具有四种状态：

  • Untracked，未跟踪，此文件没有加入到git库，不参与版本控制，可以通过git add 来使状态变为staged
  • Unmodify，文件已入库，未修改，版本库文件内容与文件中内容完全一致，该状态可以通过改变内容变为Modified，或者使用git rm移出版本库变为Untracked
  • Modified，文件已修改，git add 可以使其变为staged，git ``checkout 返回 unmodify。
  • Staged，暂存状态，执行git commit，将同步到版本库，文件变为Unmodify，执行git reset HEAD filename 取消暂存，变为Modified
  1. git add <file> ，git add .添加 所有文件到暂存区
  2. git comit -m提交暂存区所有的内容到本地仓库，-m提交信息

GIT分支

常见分支：

  • main主分支
  • dev 开发版

常用命令：

  • git branch列出所有分支
  • git branch -r远程分支
  • git branch <分支名>新建分支
  • git checkout -b <分支名>新建并切换到该分支
  • git merge <branch>合并指定分支到当前分支
  • git branch -d <branch>删除分支
  • git branch -dr <branch> 删除远程分支

分支操作

  1. git branch 查看本地分支
  2. git branch -a 查看本地线上分支
  3. git branch -vv查看本地分支与远程分支关联情况
  4. git branch name创建分支
  5. git checkout name切换到name分支
  6. git checkout -b name创建并切换到分支
  7. git branch -d name删除分支
  8. git checkout -b name origin/name创建分支并和远程分支相关联
  9. git push 远程仓库 本地分支:远程分支推送分支 简单粗暴git push 会默认push到已关联的分支，如果分支未关联，则提前需要关联(参考7或者**git push -u origin dev**)
  10. **git pull origin dev:dev**拉取远程分支的同时创建本地分支
  11. git branch --set-upstream-to=origin/<branch> 本地分支 将本地分支与远程分支相关联
  12. git -m oldBranch newBranch 重命名分支
  13. git stash存储分支修改的内容
  14. git stash pop恢复上面的内容
  15. git merge targetBranch将目标分支合并到当前分支
  16. git rebase target 将当前分支的工作移到目标分支的顶端，此操作后还需切换到target分支后再rebase一次
  17. 强制推送 git push -f origin/分支名

注意:git远程分支远程手动创建后，需要git pull拉取更新一下，才能更新到本地，本地才能查看到新建的远程分支
