小程序框架

分为逻辑层和视图层，核心是一个响应式数据绑定的框架

逻辑层

每个小程序都需要进行注册小程序和页面，其过程包含了小程序和页面的生命周期

注册小程序 App(obj)

必须在app.js中调用，且只能调用一次obj参数属性有

  • onLanunch(obj) 生命周期，监听小程序初始化，只会触发一次。参数可通过wx.getLaunchOptionsSync获取
  • onShow(obj) 生命周期 小程序启动或者从后台进入前台触发也可以使用wx.onAppShow
  • onHide() 生命周期 小程序切入后台触发 wx.onAppHide
  • onError(err) 错误监听函数 脚本错误或者API调用报错触发 wx.onError
  • onPageNotFound() 页面不存在监听函数
  • onUnhandledRejection(obj)未处理的Promise拒绝事件监听函数
  • onThemeChange(obj)监听系统主题变化
  • 其他自定义属性，可以通过this访问
App({
  onLaunch(obj){
    console.log('初始化')
    console.log(obj)
  },
  onShow(obj){
    console.log('show')
    console.log(obj)
  },
  onHide(){
    console.log('hide')
    //每隐藏一次就会增加一个监听show的监听器
    wx.onAppShow((options) => {console.log(options)})
  },
  onError(err){
    console.log('出错了！！！')
    console.log(err)
  }
})

注册页面 Page(obj)

必须在page.js中使用，响应式的数据便是在这里定义的，参数有

  • data 页面的响应式数据

生命周期函数

  • onLoad(object query) 页面加载时触发，一个页面只会调用一次，可以在参数中获取打开当前页面路径中的参数
  • onShow() 页面显示，或者切入前台时触发
  • onHide() 切入后台触发
  • onUnload() 页面卸载触发，如wx.redirectTo或wx.navigateBack到其他页面时

事件处理函数

  • onPullDownRefresh() 下拉刷新
    • 需要在app.json 中开启enablePullDownRefresh
    • wx.startPullDownRefresh可以触发下拉刷新
    • wx.stopPullDownRefresh 可以停止下拉刷新
  • onReachBottom() 上拉触底事件
    • 同样要在app.json中配置触发距离onReachBottomDistance
  • onPageScroll(obj) 滑动页面事件
    • obj为垂直方向距顶部滚动距离px
  • onAddToFavorites(obj) 用户点击收藏时触发
    • obj.webViewUrl 页面中包含web-view的url
    • 需返回一个object，用于自定义收藏内容，
      • title 自定义标题
      • imageurl 长宽比为1:1
      • query 自定义query字段
Page({
  onAddToFavorites(res) {
    // webview 页面返回 webViewUrl
    console.log('webViewUrl: ', res.webViewUrl)
    return {
      title: '自定义标题',
      imageUrl: 'http://demo.png',
      query: 'name=xxx&age=xxx',
    }
  }
})
  • onShareAppMessage(obj) 转发时触发
Page({
  onShareAppMessage() {
    const promise = new Promise(resolve => {
      setTimeout(() => {
        resolve({
          title: '自定义转发标题'
        })
      }, 2000)
    })
    return {
      title: '自定义转发标题',
      path: '/page/user?id=123',
      promise 
    }
  }
})
onShare
  • onShareTimeline() 分享朋友圈时触发
  • onTabItemTap(obj) 点击tab时触发
Page({
  onTabItemTap(item) {
    console.log(item.index)   //tab的序号，从0开始
    console.log(item.pagePath)  //点击的页面路径
    console.log(item.text)	//tabitem的按钮文字
  }
})
  • onResize(obj) 小程序屏幕旋转时触发
  • onSaveExitState() 小程序销毁前调用

此外还可以注册组件事件处理函数，如点击事件的处理函数等

页面路由

小程序是以页面栈的形式维护页面

路由方式 页面栈表现
初始化 新页面入栈
打开新页面 新页面入栈
页面重定向 当前页面出栈，新页面入栈
页面返回 页面不断出栈，直到目标返回页
Tab 切换 页面全部出栈，只留下新的 Tab 页面
重加载 页面全部出栈，只留下新的页面

模块化

可以将公共代码抽离成一个单独的js文件，作为一个模块。通过module.exports或者exports对外暴露接口。

exports 和module.exports的关系是符合CommonJS的规范的。就像nodejs里它俩的关系一���，exports是module.exports的引用

通过**require**引用模块化的文件。此外对于在app.js里配置的全局变量，通过**getApp()**获取全局的app实例。

// common.js
function sayHello(name) {
  console.log(`Hello ${name} !`)
}
function sayGoodbye(name) {
  console.log(`Goodbye ${name} !`)
}

module.exports.sayHello = sayHello
exports.sayGoodbye = sayGoodbye

var common = require('common.js')
Page({
  helloMINA: function() {
    common.sayHello('MINA')
  },
  goodbyeMINA: function() {
    common.sayGoodbye('MINA')
  }
})

// app.js
App({
  globalData: 1
})


// a.js
// The localValue can only be used in file a.js.
var localValue = 'a'
// Get the app instance.
var app = getApp()
// Get the global data and change it.
app.globalData++

wxml模板语法

https://developers.weixin.qq.com/miniprogram/dev/framework/quickstart/code.html#WXML-%E6%A8%A1%E6%9D%BF类似于vue的模板语法

常用组件(标签)

  • view 类似于div，块盒
  • text 文本，行盒
  • image 图片
    • mode ：widthFixed 宽高比固定
  • button 按钮
    • 点击事件 bindtap
  • template 模板，类似一个组件
    • is 指定 模板的类型，为自定义模板时设置的name字段
    • name 指定模板的名字，自定义模板时应有
    • data 传入模板的数据

数据绑定 {{ }}

<view> {{message}} </view>

//data 选项里定义响应式数据
Page({
  data:{
    message:'Hello MINA'
  }
})

{{}}内和vue的写法基本一致，可以进行的写法有

  • 运算，算数，逻辑，字符串，数据路径(arr[0],obj.key)
  • 组合 直接在{{}}写数组，或者对象 (省去大括号)，并且可以...解构

注意 "{{}} "有空格会被解析成字符串

<view wx:for="{{[1,2,3] + ' '}}">
  hello {{item}}
</view>
<view wx:for="{{[1,2,3]}} ">
  hello {{item}}
</view>

列表渲染 wx:for

<view wx:for="{{array}}">{{item}}</view>

Page({
  data:{
    array:[1,2,3,4,5]
  }
})

wx:for-item 自定当前的变量名，默认为**item**wx:for-index 指定当前的索引名，默认为index

<view wx:for="{{[1,2,3]}}" wx:for-item="ele" wx:for-index="idx">
  hello {{ele}} {{idx}}
</view>

wx:key 同vue一样，列表渲染默认key为索引，动态列表需要额外绑定key，对于静态列表则可以不考虑。

注意 wx:for=“ss” ，值为字符串时，会遍历每一个字符

条件渲染 wx:if

决定是否渲染某一块代码，同样可以组合wx:elif``wx:else，上面的条件满足，就不会再渲染下面的内容

<view wx:if="{{isShow}}">
  上面的显示
</view>
<view wx:else>
  下面的显示了
</view>

同vue的v-if和v-show的区别，小程序的wx:if与v-if一致，只要条件不满足，就不会渲染block wx:if，在需要一次条件判断多个组件时，可以使用包装元素block来进行条件渲染。

💡<block/> 并不是一个组件，它仅仅是一个包装元素，不会在页面中做任何渲染，只接受控制属性。

模板 template

模板可以看成是对于一段代码块的封装，提高了代码块的复用性，类似组件

  1. 定义模板

<template name=""></template> name属性为模板的名字，定义模板的必填项。

  1. 使用模板 通过is属性指定模板的name来确定改模板是哪个
<template name="one">
  <button> 1 </button>
</template>
<template name="two">
  <button> 2 </button>
</template>
<template is="one" data=""/>
<template is="two" data=""/>
  1. 上面的例子完美的诠释了代码冗余，两个模板只有1，2 不一样，怎么能够用一个模板来表示呢。小程序提供了一个data属性来给模板传递数据
<template name="num" >
  <button> {{num}} </button>
</template>
<template is="num" data="{{num:1}}"/>
<template is="num" data="{{num:2}}"/>

模板中变量的作用域只来自于传入的data，因此需要将key对应

引用 import 和 include

import 可以引用目标文件定义的template，目标文件import的template不会被引入。

<template name="item">
  <view>item</view>
</template>

<import src="./item"/>
<template is="item" data=""/>

include可以将目标文件除了 外的整个代码引入，相当于是拷贝到 include 位置，如：

<template name="item">
  <view>item</view>
</template>

<template is="item" data=""/>
<template is="item" data=""/>

<include src="./item"/>
  
<!-- <template is="item" data=""/> 会警告找不到item模板 -->

wxss 微信样式

样式写法基本与css一致，还多了一些扩展的特性。

  • 尺寸单位
  • 样式导入

尺寸单位

  • rpx 响应像素 换算公式为 1rpx=屏幕宽度px/750，这里小程序做了一个规定，规定屏幕物理像素宽度为750

样式导入

@import导入外联样式表

/** common.wxss **/
.small-p {
  padding:5px;
}

/** app.wxss **/
@import "common.wxss";
.middle-p {
  padding:15px;
}

style 与class

  • style 支持动态样式，也建议写动态样式使用style
<view style="color:{{color}};" />
  • class主要写静态样式
<view class="normal_view" />

选择器

.class .intro 选择所有拥有 class=“intro” 的组件
#id #firstname 选择拥有 id=“firstname” 的组件
element view 选择所有 view 组件
element, element view, checkbox 选择所有文档的 view 组件和所有的 checkbox 组件

WXS 小程序脚本

<wxs></wxs> 或者.wxs(通过src属性引用)文件内的代码。https://developers.weixin.qq.com/miniprogram/dev/reference/wxs/注意事项：

  1. wxs 可以在所有版本中运行
  2. 语法基本和js一致，有一些不同
  3. wxs运行环境和其他js代码是隔离的，不能调用js文件中定义的函数，也不能调用小程序的API
  4. 2.4.4版本前wxs函数不能作为组件的事件回调
  5. wxs内变量的定义只能用var
<!--wxml-->
<wxs module="m1">
  var msg = "hello world";

  module.exports.message = msg;
</wxs>

<view> {{m1.message}} </view>

wxs响应事件

注意要给回调函数加上{{}}

<wxs module="test">
  function sayHello(event,ownerInstance){
    console.log('hello world!')
    console.log(JSON.stringify(event))
    console.log(JSON.stringify(ownerInstance))

  }
  module.exports={
    sayHello:sayHello
  }
</wxs>
<button bindtap="{{test.sayHello}}">sayHello</button>
  • event 事件对象
  • ownerInstance 可以用来提升动画性能，无需向AppService(逻辑层)通信就可以动态设置样式

事件系统

分类，分为冒泡事件和非冒泡事件，bind+事件名可以绑定事件，如bindtap，如果想要阻止冒泡，可以用catch+事件名，如catchtap冒泡事件如下

类型 触发条件 最低版本
touchstart 手指触摸动作开始
touchmove 手指触摸后移动
touchcancel 手指触摸动作被打断，如来电提醒，弹窗
touchend 手指触摸动作结束
tap 手指触摸后马上离开
longpress 手指触摸后，超过350ms再离开，如果指定了事件回调函数并触发了这个事件，tap事件将不被触发 1.5.0
longtap 手指触摸后，超过350ms再离开（推荐使用 longpress 事件代替）
transitionend 会在 WXSS transition 或 wx.createAnimation 动画结束后触发
animationstart 会在一个 WXSS animation 动画开始时触发
animationiteration 会在一个 WXSS animation 一次迭代结束时触发
animationend 会在一个 WXSS animation 动画完成时触发
touchforcechange 在支持 3D Touch 的 iPhone 设备，重按时会触发 1.9.90
