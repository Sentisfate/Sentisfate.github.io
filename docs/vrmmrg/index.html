一、VueX

是什么

就是实现页面间数据共享的管理工具

1.入门

  1. 安装依赖
npm install vuex@next --save
  1. 配置store文件夹
    1. src下新建store文件夹，新建index.js
    2. 编写index.js
import Vue from 'vue';
import Vuex from 'vuex';
Vue.use(Vuex);
export default new Vuex.Store({
  // 定义全局数据
  state: {
    count: 1,
  },
  getters: {},
  mutations: {},
  actions: {},
  modules: {},
});
  1. 配置main.js
import Vue from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'

Vue.config.productionTip = false

new Vue({
  router,
  store,
  render: h => h(App)
}).$mount('#app')
  1. 使用，通过**this.$store.state.属性名**
<template>
  <div>home</div>
</template>

<script>
export default {
  name: 'HomeView',
  mounted() {
    //使用vueX的数据
    console.log(this.$store.state.count);
  },
};
</script>

<style></style>
  1. 改变状态
    1. 在组件中派发一个事件给**Vuex**的**actions**
  methods: {
    handleClick() {
      this.$store.dispatch('add');
    },
  },
  1. 在**actions**中定义相应的事件，命名要与派发的事件名要一致，**actions**中可以处理异步操作，并向**mutations**提交事件，通过**ctx.commit**
  actions: {
    add(ctx) {
      console.log('我收到了事件');
      //   ctx表示上下文,就是store
      console.log(ctx);
      ctx.state.count++;
      console.log(this);
      //commit提交事件给mutation
      ctx.commit('increment');
    },
  },
  1. 在**mutations**定义处理相应的事件
  mutations: {
    // 在mutataions中接收actions提交的方法
    increment(state) {
      //state为默认传入的形参，为store的state属性
      console.log(state);
      state.count++;
    },
  },

流程view->action->mutation->state->view

思考，为什么vueX不直接改state？

虽然可以this.$store来调用dispatch,commit,state来跳过中间的循环，为什么不直接用呢

VueX的好处

多个路由可以共享数据，除非页面刷新

二、Vue3中的使用

  1. 安装

**npm install vuex@next --save**

  1. 配置store/index.js
// 导入createStore方法用于创建vuex实例
import { createStore } from 'vuex';
// 导出vuex实例
export const store = createStore({
  state() {
    return {
      count: 0,
    };
  },
  mutations: {
    increment(state) {
      state.count++;
    },
  },
});
  1. 在main.js中将store作为插件安装
import { store } from './store';
createApp(App).use(store).mount('#app');
  1. 组合式api中访问store useStore
import { useStore } from 'vuex';
const store = useStore();
  1. 访问状态 store.state
  2. 提交事件 **store.commit(事件名)**此处mutation内必须执行同步操作，即提交后立即修改，可以写异步，但是为了debug获得每一次提交后的记录，不推荐
  3. 分发事件 **store.dispatch(事件名)** 此处action中可以为异步操作，指的是action中的为异步操作
  4. getter，对state中的状态进行过滤，派生出一些类，相当于state的全局的计算属性

三、mapState

作用，可以结合计算属性批量的从state中取值用法 : **...mapState(['state.属性1',...])**

import { mapState } from 'vuex';
export default {
  computed: {
    ...mapState(['city', 'name', 'age']),
  },
}

四、mapMutation

1.引例

 mutations: {
    handleCity(state) {
      state.city = '长沙';
    },
    increment(state) {
      state.count++;
    },
    decrement(state) {
      state.count--;
    },
  },

  methods: {
    changeCity() {
      this.$store.commit('handleCity');
    },
    increment() {
      this.$store.commit('increment');
    },
    decrement() {
      this.$store.commit('decrement');
    },
  },

代码冗余：

改进：

  methods: {
    ...mapMutations(['handleCity', 'increment', 'decrement']),
  },

五、Module

作用，拆分vuex的逻辑，提高代码的复用性

步骤：

  1. 新建modules文件夹，在其中写好module的内容
export const CountHook = {
  state: {
    count3: 1,
  },
  mutations: {
    add(state) {
      state.count3++;
    },
  },
};
  1. 在store/index.js 下引入module
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);
import { CountHook } from './modules/CountHook';
export default new Vuex.Store({
  modules: {
    CountHook,
  },
});
  1. 使用，注意此时状态在**this.$store.state.CountHook.count3**** **，mutation中的方法不会嵌套
<template>
  <div>
    <div @click="add">{{ count3 }}</div>
  </div>
</template>

<script>
  import { mapMutations, mapState } from 'vuex';

  export default {
    computed: {
      ...mapState({
        count3: (state) => state.CountHook.count3,
      }),
      // count3() {
      //   return this.$store.state.CountHook.count3;
      // },
    },
    methods: {
      ...mapMutations(['add']),
    },
  };
</script>
