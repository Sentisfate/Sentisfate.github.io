vue3中的插槽和vue2中的插槽的区别。

具名插槽

vue3中依然可以按照vue2的写法来写插槽。此外，还提供了v-slot这个指令来写插槽。

vue2写法

<template>
  <div ref="testRef">
    111
    <slot></slot>
    <slot name="juming"></slot>
  </div>
</template>

<Son>
  nihao
  <div slot="juming">
    具名插槽
  </div>
</Son>

vue3写法

<Son>
  <!-- <div slot="juming">
  具名插槽
</div> -->
  <template v-slot="juming">
    vue3写法
  </template>
  <template #juming>
    vue3写法
  </template>
  <template #default>
    vue3写法 默认插槽
  </template>
</Son>

注意两种写法不能并存，v-slot可以缩写为#

动态插槽名

  <template v-slot:[dynamicSlotName]>
    ...
  </template>

  <!-- 缩写为 -->
  <template #[dynamicSlotName]>
    ...
  </template>

作用域插槽

一般情况，插槽只能获得父组件的数据。但是有时我们需要让插槽同时获取子组件和父组件的数据。对于只有默认插槽写法如下

<Son v-slot="slotProps">
  {{ slotProps.msg /**默认插槽 */ }}

</Son>

<template>
  <div>
    111
    <slot></slot>
    <slot name="juming"></slot>

    <slot :msg="'这里是默认插槽'"></slot>
    <slot name="zuoyongyu" :msg="'这里是具名插槽'"></slot>
  </div>
</template>

默认插槽和具名插槽混合

<Son>
    <template v-slot:zuoyongyu="props1">
      {{ props1 }}
    </template>
    <template #default="{msg}">
      {{ msg }}
    </template>
</Son>

总结

  1. 插槽的写法
    1. vue3中仍能按照vue2的格式写，但是不能vue2，vue3格式混写
    2. vue3中增加了一个v-slot指令来进行插槽的匹配
  2. 作用域插槽使用
    1. 子组件中预留插槽，插槽出口处绑定相应的属性，类似组件传值中的绑定
    2. 父组件中使用 **v-slot:name="props"**来获取子组件绑定的数据，props可以解构缩写，如**#slotname="{key1,key2}"**
    3. 对于只有默认插槽，可以使用使用 **<Component v-slot="props">{{props.key}}</Component>**

<template>
  <slot name="showModalBtn" :showModal="showModal"></slot>
  <a-modal v-model:visible="visible" :title="'hello world'" @ok="handleOk">
    <slot name="content"></slot>
  </a-modal>
</template>

<script setup lang="ts">
import { ref, defineProps } from 'vue'

const visible = ref<boolean>(false);
// const props = defineProps([ 'title',])

const showModal = () => {
  visible.value = true;
};

const handleOk = (e: MouseEvent) => {
  visible.value = false;
};


</script>
