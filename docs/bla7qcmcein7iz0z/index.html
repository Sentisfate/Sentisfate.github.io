类型断言的合理前提

2023-06-21 06:59

💡什么样的类型断言才是合理的，从报错的提示来看，需要两个类型之间充分重叠，将类型看为集合的话，重叠可以看作是集合之间有交集那么这种交集是一种什么程度呢？

首先猜测是包含关系，浅显的进行探索一下

type numOrstring=number|string

function bar(x:numOrstring){
  x as number
}
function foo(x:number){
  x as numOrstring
}
//上面的都是包含关系，可以互相转换
function baz(x:number|boolean){
  x as numOrstring//相交关系也可以
}

type Gender = "男" | "女" | "其他";
type Nan="男"
type Person = {
  gender: Gender;
};
function foo(x:Nan){
  x as Gender//互相断言不会报错
}

先是简单的number|string 之间的转换，发现除包含关系之外，如果两个集合(至少一方是联合类型)有一定的交集如number|string和number|boolean则也可以进行转换。。。。好像直接否定了前面的猜测，但是这只是简单的基础类型，看下对象类型是什么样的。

interface Person {
  gender: "男" | "女" | "其他";
}

interface Student extends Person {
  school: string;
}
function foo(s: Student) {
  console.log((s as Person).gender);
}
function bar(s: Person) {
  console.log((s as Student).school);
}
foo({gender:'其他',school:'sss'})
bar({gender:'女'})//undefined

type Person={
  gender:'男'|'女'|'其他'
}

type Student=Person&{
  school:string,
}
function foo(s: Student) {
  console.log((s as Person).gender);
}
function bar(s: Person) {
  console.log((s as Student).school);
}
foo({gender:'男',school:'sss'})
bar({gender:'女'})

发现存在继承关系与交集扩展的两个类型是可以进行断言的，虽然断言后可能访问不到某些属性，但是编译器不报错。

type Gender = "男" | "女" | "其他";
type Person = {
  gender: Gender;
};
function foo(x:Person){
  x as Gender//这种会报错，很显然对象类型和字面量类型互相断言会报错即便对象类型用到了同一个类型别名
}

再来看看具有同一属性名的对象类型

type Gender = "男" | "女" | "其他";
type Dongwu = "雄" | "雌" | "公" | "母";
type Animal = {
  gender: Dongwu;
};
type Person = {
  gender: Gender;
};

function foo(x:Person){
  x as Animal
}

:::info 得到的报错原因是gender属性类型不兼容，此外值得注意的是，这里提示类型其他不可与类型Dongwu进行比较，也就是说，对于对象类型(属性名存在着包含关系，即一个对象拥有另一个对象所有的非可选属性)，会逐个比较同名属性的类型，如果有类型不兼容，则两个对象类型不能互相断言 ::: 为验证这一点，在Dongwu的联合类型后面加上一个'其他'类型，则断言就不会报错。

但是如果两个对象类型之间的属性名不存在包含关系，则会直接报错，无法断言。

type Animal = {
  gender: Dongwu;
};
type Bird = {
  name: string;//当name为可选时，可以进行断言
};
function foo(x: Animal) {
  x as Bird;//报错无法断言
}

也就是说对象类型之间的互相断言，只能允许有着包含关系。而联合类型，则只需要满足联合类型其中之一的类型存在包含关系，即可断言，如：

type Gender = "男" | "女" | "其他"|Person;
type Person = {
  // name:string;
  gender: Gender;
};
type Animal = {
  gender: Dongwu;
};
function bar(x:Gender){
  x as Animal//满足了Gender中的Person 类型兼容 Animal类型
}

简单总结： :::info

  1. 对于基础类型，如number,string等，本身不存在交集关系，无法断言
  2. 对于联合类型，断言双方满足其中任一类型存在包含关系，即可断言
  3. 对于对象类型，需要满足继承扩展关系，才可断言
  4. any与unknow 可以与任意类型进行任意断言，因此可以利用它们来把一些不可能的断言可能化，如先转成any，再转成其他类型
  5. 对于这些组合起来的复杂类型，按照上面几条判断即可 :::

如果把联合类型参与的断言堪称其中每一个类型依次进行断言来看，可以归纳为类型之间存在子集关系，尽管对于有着可选属性的可断言对象类型是否能被成为包含关系，本人持有保留意见。 回过头再看编译器报的哪个没有充分重叠的错误，指的或许是类型包括联合(类型)其中之一的类型重叠到包含在另一类型即可

只是浅显的归纳了一下断言的合理前提，好像并没有什么实际用途。。。。
