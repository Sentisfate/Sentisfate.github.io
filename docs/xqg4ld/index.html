React-hooks

2022-07-18 08:41

一、是什么

js 是一个函数式的编程，ES6后 class 是才出现的语法糖，编码成本较高React class组件存在不足

  1. 明白面向对象class
  2. 关心this指向问题
  3. bind
  4. 代码逻辑比较混乱

为了解决类组件的不足问题，react-hooks由此诞生(2018 react16.8)。

react-hooks 使用了一些函数，将react的特性(state，lifestyle)注入到函数组件里面。

二、优势

使用函数式组件可以实现class组件的所有功能

三、钩子函数

1.useState

作用：让函数式组件具有状态

import './App.css';
import React, { useState } from 'react';

const App = () => {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
};

export default App;

useState详解，

  • 参数是状态的默认值，不给用undefined
  • 返回值是一个数组，有两个元素，第一个元素是状态，第二个元素是改变状态的函数
    • 改变状态的函数会将接收的参数赋值给状态，也就是说一个状态对应一个函数

2.hooks遍历数组

  const [list, setList] = useState(['html', 'css', 'js']);
  const Item = () => {
    return list.map((item) => <div key={item}>{item}</div>);
  };

点击删除，存在异步更新问题state 更新了，视图没有更新,❓**原因是引用类型比较的是引用的变化，引用变化会进行更新**如何解决：改变引用类型的地址

  1. 解构赋值 **setList([...list]);**
  2. 深拷贝 **setList(JSON.parse(JSON.stringify(list)))**

4.useEffect(callback，[])

默认充当于mounted和updated两个生命周期

  const [count, setCount] = useState(0);
  const [city, setCity] = useState('武汉');
  //只相当于mounted生命周期
  useEffect(() => {
    console.log('页面加载');
  }, []);
//   只相当于update生命周期
  useEffect(() => {
    console.log(city);
  }, [city]);

接收两个参数，第一个为回调函数，触发时执行第二个为仅执行更新周期的**状态数组**，只在该状态更新时触发回调函数
