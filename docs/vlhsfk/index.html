性能优化

2022-09-26 08:21

指标

加载速度，TTFB第一个请求响应时间，首次渲染，页面加载时间，DNS解析耗时，TCP连接耗时，SSL连接耗时

前端性能优化的手段

加载优化

  1. 减少http请求，合并css和js，使用精灵图等
  2. 缓存资源：使用缓存来减少请求次数，减少加载时间，服务端设置长缓存
  3. 压缩代码：服务端设置GZip
  4. 无阻塞，头部的内联样式和脚本会阻塞页面的渲染，样式放在头部使用link方式引入，脚本放在尾部使用异步加载？？
  5. 首屏加载优化，如何优化？按需加载，懒加载，滚屏加载，预加载(使用loading，资源加载完成再显示页面)等
  6. 减少cookie，静态域名不使用cookie
  7. 避免重定向
  8. 异步加载第三方资源

执行优化

  • css写在头部link引入，js写在尾部并异步
  • 避免img，iframe的src为空，空src会重新加载当前页面

iframe 允许将另一个html页面嵌入进来

  • 减少重绘和回流
