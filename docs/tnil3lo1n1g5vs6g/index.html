变量类型注释

语法：const/let variable: Type=value

函数类型注释

参数与返回值

语法：

function foo(param1: Type1,...):returnType{
  ...
}

ts会检查函数的参数和返回值，无返回值时未void

function foo(num:number):number{
  return num*2
}
function bar():void{
  
}

匿名函数

作为某一调用函数参数的匿名函数，其参数以及返回值是依据调用函数的类型进行注释的。

基础类型

js中有7个primitive data type，在typescript都有对应的类型

  • string
  • number
  • bigint
  • boolean
  • undefined
  • symbol
  • null

any

any作为一个特殊类型，表示不希望对某一个值进行类型检查，该值可以像js中一样任意使用，编译时不报错。

引用类型

数组

对数组定义类型有一下几种方式：

  • Type[]
  • any[]
  • Array
const nums:number[]=[1,2]
const anys:any[]=[1,'a',true]
const strs:Array<string>=['a']
// const nums2:number[]=[1,'2']不能将类型“string”分配给类型“number”

对象

对于对象类型的定义，须列出属性及类型，下面是一种字面量对象类型的定义方式可选属性，对应属性采用**?:**注释类型

const obj:{
  name:string,
  age?:number
}={
  name:'ssss',
  age:12
}

联合类型

某一值的类型可能会是多个类型中的一个，如可能是数字或或字符串，则可以用联合类型。使用|连接两个类型

boolean 其实就是 true|false 的联合类型

let s:number|string=1
s='1'

字面类型

可以用一系列确定的值来定义一个类型，则该类型只允许接收其中之一的值

type num='1'|'2'|'3'

使用联合类型的问题—类型收窄

无法使用只在其中之一类型上的属性或方法。

需要进行类型收窄，即确定是某一类型后再使用

  function foo(p:number|string){
    if(typeof p ==='string')
    return p.toUpperCase()
  }

常用的类型收窄方法有：

export class Narrow {
  static isNumber = (val: unknown): val is number => typeof val === 'number';
  static isString = (val: unknown): val is string => typeof val === 'string';
  static isSymbol = (val: unknown): val is symbol => typeof val === 'symbol';
  static isFunction = (val: unknown): val is Function =>
    typeof val === 'function';
  static isObject = (val: unknown): val is Record<any, any> =>
    val !== null && typeof val === 'object';
  static isPromise = <T = any>(val: unknown): val is Promise<T> =>
    this.isObject(val) && this.isFunction(val.catch);
  static objectToString = Object.prototype.toString;
  static toTypeString = (val: unknown): string => this.objectToString.call(val);
  static isPlainObject = (val: unknown): val is object =>
    this.toTypeString(val) === '[object Object]';
}

类型别名

可以看成是类型变量，方便使用一些复杂类型。语法：type TypeName=Type

type Point = {
  x: number;
  y: number;
};

注意：别名只是别名，本质类型是一样的，不会因为别名不一样导致类型不一样(前提两个别名对应的类型一致)，如：

type T1=number|string
function foo(t:T1):number|string{
  return t//T1 就是 number|string 并不会报错
}

接口

接口是对象类型声明的另一种方式，其同样只是一种别名，ts只关心其本质类型的区别语法：

interface Point {
  x: number;//这里,和;都不会报错
  y: number;
}

接口与类型别名的区别

接口能做到的功能type几乎都能做到，区别在于type 无法重新打开类型添加新属性，而接口始终可以扩展。此外type可以声明联合类型，interface只能声明对象类型

interface Person{
  gender:'男'|'女'|'其他'
}

interface Student extends Person{
  school:string
}

const xiaoming:Student={
  gender:'其他',
  school:'xxxxxschool'
}

type Person={
  gender:'男'|'女'|'其他'
}

type Student=Person&{
  school:string,
}
const xiaohong:Student={
  gender:'其他',
  school:''
}

接口重新打开添加属性

interface Person {
  gender: "男" | "女" | "其他";
}

interface Student extends Person {
  school: string;
}
interface Student {
  name: string;
}
//即便是后面继续添加父接口字段，也会影响子接口
interface Person {
  age: number;
}
const xiaoming: Student = {
  gender: "其他",
  school: "xxxxxschool",
  name: "xiaoming",
  age: 12,
};

type Person={
  gender:'男'|'女'|'其他'
}
type Person={
  age:number
}

//报错 标识符 Person 重复

类型断言

类型断言用来解决TS无法知道值的类型信息的情况，可以人为的进行类型断定，告知编译器该值为和类型，但前提要是合理的。语法：value as Type或者 value

function foo(x:number|string){
  // return (<string>x).toUpperCase()
  return (x as string).toLowerCase()
}

前面也谈到了类型断言的前提必须是合理的，像下图中这种断言就会报错。

类型推断

当在声明变量的时候，如果没有类型声明，ts会根据等式右侧的值自动为变量进行类型推断。

同样适用于其他类型，并且在你尝试进行赋值的时候，类型不对就会进行报错。

null 和undefined

null和undefined取决于是否启用strictNullChecks选项，如果启用，则需要进行类型收窄检查值是否为undefined，否则会报错。

function foo(x:null|undefined|string){
  x?.toUpperCase()//?.可以完美测试是否为空值
}

?与！

？表示值可能为空，不为空时进行后面的操作!则是非空断言符，表示告诉编译器该值一定不为空，编译不报错，但是运行时可能会报错，由值是否为空决定。

function foo(x:null|undefined|string){
  x?.toUpperCase()//等价于下面
  if(x){
    x.toLocaleLowerCase()
  }
  x!.toLocaleLowerCase()//直接告诉编译器该值不为空，不会改变运行时的行为，该报错还得报
}
