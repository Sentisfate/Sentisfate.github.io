划分为k个相等的子集

2022-09-20 03:51

题目描述

解法1 回溯+剪枝

思路：

目标是要由k个集合，k个集合和相等，不妨记为m，先排除掉一些不可能的情况，假设nums必定能分成k份，每个元素都必定会在某个集合中，每个集合都必定有一个或多个集合。因此可以从下面两个角度来考虑。

一、元素视角

每个元素在选择集合的时候，都有k种选择。即进哪一个集合，集合存在着和为m的限定。则存在着以下的限定条件。集合目前的和s[j]，当前元素的大小nums[i]剪枝条件：

如果上式不满足，则进入下一个集合进行判定。如果满足，就进行s[j]+=nums[i]。接下来对下个元素进行上述判断，如果满足集合条件，就进入集合，一个集合都无法进入的话，就回溯到上一个元素，让其选择下一个集合。 直到所有元素都进行了选择后，可以确定是否有解。 以上过程虽能求出解，但是时间复杂度较高，为了能让(1)的情况更容易发生，也就能提高时间效率，减少递归次数。因此提出的第一个优化手段是将数组进行排序。 优化2.当当前集合与上一个集合的和相等时，此时当前元素已经是不满足上一个元素了，有两种情况， 1. 和不满足 2. 递归不满足，剩下的集合有 … dp[i-1], dp[i] … 对于其他元素而言，这两次的选择的区别就是其他集合和dp[i-1]或者dp[i],但是这两个集合相等，就表示没有区别，因此就都不满足 优化3. 对于第一个元素而言，如果大于m,说明必定不能划分。

临界条件： :::info

::: 证明： :::tips

:::

/**
 * @param {number[]} nums
 * @param {number} k
 * @return {boolean}
 */
var canPartitionKSubsets = function (nums, k) {
  // 思路，总和/k便是每个子集的和，接下来划分即可
  let sum = nums.reduce((pre, cur) => pre + cur);
  let avg = sum / k;
  if (avg % 1) {
    return false;
  }
  nums.sort((a, b) => b - a);
  // 第一个元素大于集合的和
  if (nums[0] > avg) {
    return false;
  }
  //定义一个数组来记录每个集合的和
  let dp = new Array(k).fill(0);
  //递归的遍历数组
  const backtrack = (nums, index, dp, k, avg) => {
    // 结束条件，所有的元素都做了选择
    if (index >= nums.length) {
      // 这里一定会为true，因为所有元素都做了选择
      return true;
    }
    // 对当前的元素进行选择
    for (let i = 0; i < k; i++) {
      //当当前集合与上一个集合的和相等时，此时当前元素已经是不满足上一个元素了，有两种情况，
      // 1.和不满足
      // 2.递归不满足，剩下的集合有 ... dp[i-1], dp[i] ... 对于其他元素而言，这两次的选择的区别就是其他集合和dp[i-1]或者dp[i],但是这两个集合相等，就表示没有区别，因此就都不满足
      if (i > 0 && dp[i - 1] == dp[i]) continue;

      //当超出规定的和时，选择下个集合
      if (dp[i] + nums[index] > avg) continue;
      // 将当前元素进入集合
      dp[i] += nums[index];

      //对下一个元素进行选择,这一步进行了递归，如果最后一个满足的话，就都满足
      if (backtrack(nums, index + 1, dp, k, avg)) return true;
      // 否则，将当前的元素从集合中撤出，选择下一个集合,回溯的关键所在
      dp[i] -= nums[index];
    }
    //k个集合都不满足就返回false
    return false;
  };
};
