实现类似于vue-router的路由跳转

一、安装 v5

pnpm i react-router-dom@5

二、router v6版本以下的使用方法

  1. 导入相关模块
import { BrowserRouter as Router, Route, Link } from 'react-router-dom';
  1. 定义路由规则 Link
{/* 使用link组件配置路由入口 */}
<Link to={'/'}>首页</Link>
<Link to={'/articles'}>列表页</Link>
<Link to={'/details'}>详情页</Link>
  1. 定义路由出口 Route exact 精确匹配，默认模糊匹配，只匹配前面的字符，符合就进路由
<Route path={'/'} exact component={Home}></Route>
<Route path={'/articles'} component={ArticlesList}></Route>
<Route path={'/details'} component={Details}></Route>

常用组件

  • Router组件，包裹整个应用，一个React应用只需要使用一次
  • HashRouter，使用url的hash值实现，如 …/#/details
  • BrowserRouter 使用H5的historyAPI实现，样子上省略了# 如…/details
  • Link组件，导航链接a标签
    • to:string|object
      • string：链接位置的路由，包括路径名，查询参数，等
      • object：
        • pathname 表示要链接到的路径的字符串
        • search：表示查询参数的字符串形式
        • hash 哈希，表示放入网址的hash，例如#a-hash
        • state 状态持续到 location，子组件可以通过**props.location.state**获取，其余属性也同样在location中可以获取
<Link to={{
    pathname: '/courses',
    search: '?sort=name',
    hash: '#the-hash',
    state: { msg: 'hello state' }
}}/>

  • replace:bool 为true则路由跳转时，会把当前的路由出栈，然后再进栈，不能通过返回来回到原来的路由
<Link to={toObj} replace>
    first
</Link>
  • innerRef:function 允许访问当前Link最终被渲染成的Dom对象
<Link to={toObj} replace innerRef={(node) => console.log(node)}>
    first
</Link>

  • others 如className，ID
  • Switch 内部的路由出口，匹配成功后便不会再执行后面的路由出口。不放在里面，匹配成功的都会渲染。
  • **NavLink** 当它与URL匹配时，可以为其渲染元素样式添加样式属性，
    • **activeClassName**: string 默认是active，可以改为其他
    • **activeStyle**: object， **activeStyle={{fontWeight: 'bold',color: 'red'}}**
    • **exact** 仅在pathname完全匹配时才会赋予样式，尾部斜线有无都会赋予样式
    • **strict** 默认为false，为true时会考虑pathname尾部的斜线
    • isActive:func(match，location) 为了确定链接是否处于活动状态而添加的额外逻辑的函数，不仅能够验证链接的路径名，函数返回一个bool值，为true则渲染样式，否则不给样式
    • location :object 比较不同的位置
import React from 'react';
import {
  BrowserRouter as Router,

  NavLink,
  Switch,

const App = () => {
  function oneLevelRoute() {
    let history = useHistory();
    console.log(history);
    
    return <div>{history.location.pathname}</div>;
  }
  return (
    <div>
      <NavLink exact to={'/'}>
        home
      </NavLink>
      <NavLink
        to={'/about/123'}
        activeClassName={'active'}
        isActive={(match, location) => {
          console.log('match...');
          console.log(match);
          console.log(location);
          if (!match) {
            return false;
          }
          console.log(match.url.split('/')[match.url.split('/').length - 1]);
          if (match.url.split('/')[match.url.split('/').length - 1] % 2 == 0) {
            return true;
          }
          return false;
        }}
        >
        
        <Switch>
          <Route path={'/'} component={oneLevelRoute}></Route>
          <Route path={'/about'} component={oneLevelRoute}></Route>
        </Switch>
        </div>
      );
      };
      
      export default App;

  • **Redirect**** 重定向**

    • push:bool 为true会将新地址推入history，默认为false，会覆盖当前的history中的地址
    • from:string 要重定向的地址，有该属性时，要放到Switch里。

  function oneLevelRoute() {
    let history = useHistory();

编程式导航

通过Route渲染的组件中，可以通过props.history 来进行编程式跳转

URL Parameters

依赖模块 **useParams**** **这个函数可以获取地址栏后拼接的请求参数使用：路由出口对应的组件中 **let {id}=useParams()**

  1. 导入依赖
import {
  BrowserRouter as Router,
  Switch,
  Route,
  Link,
  useParams
} from "react-router-dom";
  1. 定义路由规则和路由出口
<Link to={'/home'}>首页</Link>
<Link to={'/articles'}>列表页</Link>
<Link to={'/details'}>详情页</Link>
<Switch>
  <Route path={'/:id'} component={UserId}></Route>
</Switch>
  1. 定义子组件，并在子组件中获取请求参数。
function UserId() {
  let { id } = useParams();
  return <div>id:{id}</div>;
}

useRouteMatch

作用，获取匹配的前端路由，可用于子路由的匹配。使用

  1. 导入依赖
import {
  BrowserRouter as Router,
  Route,
  Link,
  Switch,
  useParams,
  useRouteMatch,
} from 'react-router-dom';
  1. 定义一级路由及出口，注意有二级路由的路由出口，不能用exact精确匹配
render() {
    function Home() {
      return <div>home</div>;
    }
    function About() {
      return <div>about</div>;
    }
    function Topics() {
      return <div>topics</div>;
    }
    return (
      <Router>
        <div>
          <Link to={'/'}>home</Link>
          <Link to={'/about'}>about</Link>
          <Link to={'/topics'}>topics</Link>

          <Switch>
            <Route exact path={'/'} component={Home}></Route>
            <Route exact path={'/about'} component={About}></Route>
            <Route path={'/topics'} component={Topics}></Route>
          </Switch>
        </div>
      </Router>
    );
  }
  1. 在一级路由里定义二级路由规则
function Topic() {
    console.log(useRouteMatch());
    return <div>{useRouteMatch().url}</div>;
}
function Topics() {
    console.log(useRouteMatch());
    let { url, path } = useRouteMatch();
    return (
        <div>
            topics
            <div>
                <Link to={url + '/first'}>first</Link>
                <Link to={url + '/second'}>second</Link>
                <Switch>
                    <Route path={path + '/first'} component={Topic}></Route>
                    <Route path={path + '/second'} component={Topic}></Route>
                </Switch>
            </div>
        </div>
    );
}

关于useRouteMatch的返回值

path和url的区别url是浏览器上的地址栏后面的路径，是一个实际存在的路径path则是一个可以匹配的路径，附带参数或变量的路径

三、react-router-dom v6

1.安装

**pnpm i react-router-dom@6**

2.入门

  1. 案例
import { BrowserRouter as Router, Routes, Route } from 'react-router-dom';
const App = () => {
  function Home() {
    return <div>home</div>;
  }
  function About() {
    return <div>About</div>;
  }
  return (
    <div>
      hhh
      <Routes>
        <Route path="/" element={<Home />}></Route>
        <Route path="/about" element={<About />}></Route>
      </Routes>
    </div>
  );
};

在**routerv6**版本，**Route**要在**Routes**标签下，且同样，一次匹配成功只能匹配最前面的一个。**v5**用的是**component={Component}** ，v6用的是**element={<Component/>}**此外，Route的path属性可以写相对路径，可以写嵌套路由。

4.组件详解

1.Link

属性

  • to string|obj 可以写��对路径，obj取值和v5一样，state需要单独在外面定义
  • state，路由对应的组件的props不再接收history，获取state要通过**useLocation**来获取。
<Link
    // exact={true}
    to="/"
    state={{ msg: 'hello home' }}
    // to={{
    //   pathname: '/',
    //   state需要在外面单独定义
    //   state: { msg: 'string' },
    //   hash: '#abcd',
    // }}
    >
 <Route path="/" element={<Home msg="hello"></Home>}></Route>

function Home(props) {
    let location = useLocation();
    console.log(location);
    console.log('homeprops:', props);
    return <div>home</div>;
}

2.NavLink

属性移出了原来的activeClassName和activeStyle，使用className和style来代替

  • **className:fun** 该函数接收一个对象作为参数如**{isActive}**这个对象有一个isActive属性，为bool值，用于判断其是否处于激活状态，返回样式名
  • **style ：function** 同上，也是接收一个函数，不同的是它要返回一个style：obj
<NavLink
    to={'first'}
    style={({ isActive }) => {
        console.log('first');
        const style = {
            color: 'blue',
            background: 'black',
        };
        return isActive ? style : undefined;
    }}
    >
    first
</NavLink>
<NavLink
    to={'second'}
    className={(obj) => {
        return obj.isActive ? 'active' : '';
    }}
    >
    second
</NavLink>

3.Route

属性

  • path 可以写相对路径，省去了useRouteMatch，注意，值为*时表示匹配没有定义过的路由出口，也叫404
  • element 优化了原来v5的component，可以直接在组件中自定义属性传值，如:
<Route path="/" element={<Home msg="hello"></Home>}></Route>

4.Routes

替换了v5的switch ，并且Route都得包在Routes里。

5.Hooks

1.useNavigate替换useHistory实现路由跳转

操作一样，在函数式组件中使用

import { useNavigate } from "react-router-dom";

function MyButton() {
  let navigate = useNavigate();
  function handleClick() {
    navigate("/home");
  }
  return <button onClick={handleClick}>跳转至home</button>;
}
