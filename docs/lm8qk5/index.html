Mysql配置的提取

目的

mysql的配置会经常会动，提取出来，方便以后操作。还可以增加一个debug属性，boolean型，用于调试，显示错误信息。

做法

在后端文件夹下创建一个config文件夹，将mysql配置剪切到index.js下，增加一个debug:true,用于调试，然后导出。在db文件夹下的index.js导入mysql配置并使用。

module.exports = {
    debug: true,
    mysql: {
        host: 'localhost',
        port: 3306,
        user: 'root',
        password: '1234',
        database: 'db_todolist'
        // timezone:'SYSTEM' 处理时区问题
    }
}

//引用
const { mysql: config } = require('../config');
// 创建连接
const con = mysql.createConnection(config)

异常处理 try-catch

throw抛出异常，一般是抛出一个error报错对象，如果不处理的话，程序会停止运行，后面的代码就无法执行。try-catch可以让程序出了异常后，后面的代码一样能够正常运行，增加了程序的健壮度。语法

try{
    可能会出异常的代码
}catch(error){
    处理异常的对象的代码
}

代码中的使用

const {debug}=require('../config/index')

try{
    const data= await getAll(sql)
    res.send({
        code:0,
        msg:'成功',
        result:data
    })
}catch(error){
    res.send({
        code:100101,
        message:'获取所有待办失败',
        result:debug?error:''
    })
}
