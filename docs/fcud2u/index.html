Vue-cli 前端工程化

一、介绍

1.是什么

vue-cli也叫做vue脚手架，它是一个基于vue.js帮助快速开发项目的工具。

2.安装

打开cmd，或者终端，输入下面的代码进行全局安装

npm i @vue/cli -g

安装完成后，如果没出错的话，可以使用如下代码进行测试是否安装成功

vue --version

成功的话会输出对应的脚手架的版本号。

如果网络问题导致无法安装成功，可以尝试使用**yarm**来安装。

npm i yarm -g

再使用yarm来安装vue-cli

yarm global add @vue/cli

二、搭建项目

  1. 创建项目根目录，在根目录处打开cmd或终端或powershell，此处以vscode的终端为例。

  1. 在根目录处打开cmd或终端或powershell

  1. 在终端里输入**vue create 项目名**后回车开始搭建项目

  1. 进入到下面的选择页面，通过方向键可选择，选项，这里我们选取手动选择配置要素，并回车。

  1. 接着进入到选择要配置到哪些要素，这里其他要素暂不需要，只选**Babel**，回车

  1. 接着是vue版本配置选择，根据vue的版本选择合适的，这里我选择的是2.x版本。回车

  1. 接着是配置文件是以什么形式存储，我们选择package.json。回车

  1. 接着是是否保存此次配置选择，可作为下次项目的预设。这里我选择**y**即保存此次配置。回车。

  1. 上一选项为y时，会有这次的选项，即此次预设保存为什么名字。这里根据情况输入这次的配置预设名字就行。

  1. 回车后就会自动开始搭建项目了。

  1. 搭建成功后，可根据提示运行脚本，以确定是否搭建成功。首先是进入到项目的文件夹下，这里步骤 **3 **中我们创建的时候，项目名为todo，所以，创建了todo文件夹作为项目的文件夹，在终端输入**cd todo **进入到todo文件夹下。
  2. 输入**npm run serve**启动服务。

  1. ctrl加鼠标点击点击上图中的链接，进入到搭建好的vue项目的主页。至此项目搭建完成。

三、项目目录结构

1.目录结构

在开始项目之前，先来研究一下vue脚手架搭建的项目的目录结构

首先是根目录，这个目录名是依据创建项目时**vue create XXX**决定的。接着是node_modules文件夹，这个文件夹里存放的是项目依赖的工具包。下载的包都会存放到这里。其次是public文件夹，存放的是html文件，是vue默认创建的主页，和相应的图标。

src文件夹是存放代码的文件夹，其下有默认的两个文件夹，assets用于存放要用到的资源如图片等，components文件存放的是组件，将来自定义的单文件式组件都是写在这里。此外APP.vue文件是脚手架自动生成的组件，相当于一个根组件，其他组件都可以挂载到它身上。components中的组件可以挂载到其身上。main.js实现的是组件挂载到index页面上。

package.json，此文件就不多说了吧，项目的配置文件,它是这个项目的清单,是项目的核心,包含程序或包的名称,版本号,依赖项列表,脚本等.具体由其中的属性决定.

2.启动流程

如果还记得前面的测试是否搭建成功的操作的话，应该记得下面这句命令来启动web服务。**npm run serve**本质上是执行了package.json的脚本。

随后就可以通过url来访问页面了。研究下index.html，主页里有下面这么一句注释，**<!-- built files will be auto injected -->**，翻译下来就是打包文件会自动注入，并没有任何的引用。那么我们的组件是怎么导入到这个页面的呢。

访问主页并打开开开发者模式，浏览源代码，发现多了两句引用。

这两句干了什么？怎么来的？仔细看看这两句引用，引用的都是js文件夹��的文件，等等，好像我们的项目目录结构里并没有js文件夹。往左一看，果然目录结构发生了大变。

造成这一现象的原因就是**webpack **这个模块打包工具。

**Webpack **是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静态分析，然后将这些模块按照指定的规则生成对应的静态资源。

那么原因也就明了了，webpack对我们的项目文件结构分析后进行了打包，并自动引入了main.js这个入口文件。再通过main.js引入跟组件App.vue，其余子组件则绑定在App上一并引入进来。

四、基本示例

1创建组件

在components文件夹下新建一个组件文件，**UserCom.vue**注意，根据风格指南，组件名要是pascal命名规则**，且要有多个单词组成为了防止与未来的html元素冲突，因为html元素基本都是单个单词。

接着编写组件代码，一个组件是html，css ，js的集合**，因此在这个文件里可以写这三种代码。首先是html代码，也就是组件的模板。

<template>
	<!--  组件模板部分 -->
	<div>我是子组件</div>
</template>

接着是js代码，也就是，这个文件返回的组件对象。

<script>
	export default {
		name: 'UserCom', //组件名为多个单词，尽可能与文件名一致
	};
</script>

最后是组件的css样式。

<style scoped>

</style>

2.使用组件

将创建好的组件挂载到App.vue上，需要以��几步。

  1. 导入组件对象
import UserCom from './components/UserCom.vue';
  1. 挂载组件
export default {
  name: 'App',
  components: {
    HelloWorld,
    UserCom,
  },
};

这里的**export default** 是ES6语法，相当于导出了一个组件对象，因此在这个组件对象**components**选项里可以挂载子组件。

  1. 引用组件
<template>
  <div id="app">
    <img alt="Vue logo" src="./assets/logo.png" />
    <HelloWorld msg="Welcome to Your Vue.js App" />
    <user-com></user-com>
  </div>
</template>

在这里没有使用原来的组件名**UserCom**而是使用了**user-com**是因为html里并不区分大小写，且都推荐使用全小写。因此为了区分大小写而使用-符号。

2.2子向父传值

  1. 子组件定义要提交的数据
data() {
	return {
		todo: { id: 1, content: '' },
	};
 },
  1. 子组件通过**$emit**触发自定义事件和提交数据。
  methods: {
    submit() {
      this.$emit('updateList', this.todo);
      console.log('enter is downup...');
    },
  },
  1. 父组件绑定（或者叫做监听）自定义事件
<todo-item @updateList="updateList"></todo-item>
  1. 父组件处理自定义事件
  data() {
    return {
      lists: [{ id: 0, content: 'todo  0' }],
    };
  },
  methods: {
    updateList(item) {
      // 如果数组有元素的话，就修改传入的id值(默认为1)
      if (this.lists[0]) {
        this.lists.push({
          id: this.lists[this.lists.length - 1].id + 1,
          content: item.content,
        });
      } else {
        //没有的话，就不修改id
        this.lists.push({
          id: item.id,
          content: item.content,
        });
      }
    },
  },
