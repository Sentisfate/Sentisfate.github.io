无状态组件

本质是一个箭头函数，返回的是个dom，没有this，没有state数据，但可以接收父组件的传来的props，通过形参来接收。

<Son name="父组件传值"></Son>

import React, { Component } from 'react';

// 无状态组件接收父组间的数据是通过函数传参props(形参)来获得
const Son = (props) => {
  return <h1>Son{props.name}</h1>;
};

export default Son;

渲染富文本

有一个固定的格式

  1. 获取富文本
  2. 将富文本接收到一个对象的__html属性中，只能是__html
  3. dangerouslySetInnerHTML={对象}
import React, { Component } from 'react';

class Index extends Component {
  render() {
    const rawHtml = '<h2>富文本<i>斜体</i></h2>'
    const rawHtmlData = {
      __html:rawHtml  //格式必须是这样的__html
    }
    return (<div>
       <p dangerouslySetInnerHTML={rawHtmlData}></p>
    </div>)

  }
}
export default Index;

组件懒加载

作用：前端项目基本都采用打包技术，如 Webpack，随着我们引用的第三方库越来越多或业务逻辑代码也越来越复杂，相应打包好的 bundle.js 文件体积就会越来越大，因为需要先请求加载资源之后，才会渲染页面。懒加载解决了这样的一个问题，即让浏览器按需加载。使用步骤：

  1. 通过**React.lazy()**方式引入组件
const Son2 = React.lazy(() => import('./Son2'));
  1. 通过**React.Suspense**来使用组件
<React.Suspense fallback={<div>Loading...</div>}>
  <Son2></Son2>
</React.Suspense>

PureComponent

是用来解决子组件重复渲染，提高性能的一个类。让子组件继承该类，在与父组件没有关联的时候，不会随着父组件渲染时一起渲染。

import React, { PureComponent } from 'react';

// 防止子组件没有关联也刷新
// 使用PureComponent
class Son extends PureComponent {

    // 内置函数 
    // 返回一个布尔值 
    // 决定一个渲染周期中是否render
    // shouldComponentUpdate(){
    //     // 太武断了
    //     // 有联系了还不变
    //     return false
    // }

    render() {
        console.log('儿子刷新了')
        return (
            <div style={{marginTop:'50px'}}>
                子组件的计数器
            </div>
        );
    }
}

export default Son;

import styles from './App.module.css';
import React, { Component } from 'react';
import Son from './Son';

class App extends Component {
  state = {
    count: 0,
  };

  addNumber = () => {
    this.setState({
      count: this.state.count + 1,
    });
  };

  // 执行了 n (setState次数) + 1 (初次渲染)
  render() {
    // console.log('我刷新了')
    return (
      <div className={styles.item}>
        我是父组件
        <div onClick={this.addNumber}>计数为{this.state.count}</div>
        <Son></Son>
      </div>
    );
  }
}

export default App;

                document.body  //DOM节点

        );
    }
  1. 父组件中调用子组件

受控组件与非受控组件

受控组件是，其值或某一或某些属性受到state状态影响的组件非受控组件是属性不受state影响的组件。下面来看一组示例

state = {
  msg: '',
};  
getValue(ref) {
  console.log(ref.value);
}
...
  <input
    ref={(r) => (this.input1 = r)}
    type="text"
    value={this.state.msg}
    onChange={this.changeMsg}
    />
  <button onClick={() => this.getValue(this.input1)}>
  受控组件获取
  </button>

获取该组件相关值的方式有两种，1是通过获取state属性，2是通过ref获取dom再看非受控组件

<input
    type="text"
    ref={(r) => (this.input2 = r)}
    defaultValue={'我是默认值'}
    />
<button onClick={() => this.getValue(this.input2)}>非受控组件</button>

获取该组件的相关属性只能通过ref来获取dom，此外，该组件(input)设置默认值可以通过defaultValue，而受控组件要设置默认值则是通过state属性。

受控组件与非受控组件的区别：

  • 改变相关属性和值的机制不同，受控是setState，非受控则是可以直接根据dom内置的规则修改，从这点来看，非受控更加灵活简洁。
  • 从获取来看，受控属性有两种获取方式，从state获取相较于获取dom会更方便，这点强于非受控组件。
