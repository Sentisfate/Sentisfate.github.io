一、vue简介

1.什么是vue

:::info vue是一个动态的渐进式的构建用户界面的js框架，遵循MVVM模式 :::

  • 构建用户界面：将数据转换成界面
  • 渐进式：由简到难，自底向上慢慢改进
    • 替代jquery，js函数库功能
    • 应用组件化，重构某一页面
    • 全家桶开发某一大型应用
  • MVVM 模式，实现的响应式的框架

2.Vue的特点

  1. 数据驱动
  2. 声明式
  3. 组件化开发

二、vue的快速上手

1.环境搭建

1.1初始化

npm init -y创建一个package.json文件

1.2安装Vue

终端执行命令npm i vue@2 --save 可以简写为npm i vue@2这里vue的默认版本为3，如果要下vue2的话，用@指定版本号。

:::info 💡 vue的各种版本说明

  • vue.js是完整版，在初学阶段使用。
  • common，使用的是commonJS规范导入导出module.exports的，这种文件不能再浏览器里使用
  • esm(ES Module) 使用ES模块的规范导入导出，export default
  • runtime 运行时版本，相对于编译版本体积更小，效率更高
  • min 压缩版 :::

1.3 小结

:::success vue上手步骤

  1. 项目初始化 npm init -y
  2. 安装 npm i vue@版本 :::

2.入门案例

2.1 引入vue.js

在html 头部通过script 来引入vue.js

<script src="./node_modules/vue/dist/vue.js"></script>

2.2编写页面

在body里写一个div，id为app，所有视图将在这部分渲染，其中的{{msg}}是模板语法

<div id="app">
    {{msg}}
</div>

2.3编写vue实例

在body的最末尾，使用<script>编写vue实例。

<script>
    const vm=new Vue({
        el:'#app',//对应操作的元素，必填参数
        data:{
        },//数据部分
        method:{}
    })
</script>

语法：

  • new Vue(一个配置对象)创建)，属性有：
  • el ，表示操作的元素，值为选择器字符串，通常使用id选择器
  • data，表示数据，值为一个json对象，对象的成员名对应着元素标签体里的{{XXX}}，即要用啥声明啥，声明式特点
  • methods，方法

2.4渲染数据

在data中定义数据

<script>
    const vm=new Vue({
        el:'#app',//对应操作的元素，必填参数
        data:{
            msg:'hello world'
        },//数据部分
        method:{}
    })
</script>

以键值对形式渲染。其中msg相当于变量名，'heelo world'相当于变量值。

三、MVVM模型

1.是什么

:::info M Model(数据模型) V View(视图) 和 VM (View Model)组成 :::

1.1 核心思想

实现数据和视图的**双向绑定 **

  • Model 操作数据
  • View 显示数据
  • VM 双向绑定M和V，即操作数据后显示数据改变，操作视图改变数据后，model里的数据发生变化。VM封装了这一过程。

四、Vue实例

vue实例通常由三个配置选项组成，分别为el，data，methods

1.el

el配置选项是用来确定vue实例所关联的视图。其取值可以是选择器字符串或者dom对象。el: string|elemet注意点： :::info

  • 只会关联string所匹配的第一个元素。
  • 只能匹配一个元素，因此推荐使用id选择器。
  • 使用选择器相较于dom元素多数情况更方便些，更推荐 :::

2.data

通过data选项配置的各种数据叫属性，也叫**状态。**以键值对的形式，

data:{
  msg:'111',
}

上面的msg就可以看成是变量名。可以通过Mustache语法{{}}使用。如{{msg}}msg的 使用，遵循着js语法，如对象的话可以通过对象.属性来获取属性数据，数组的话可以通过索引获取某一个元素等。

3.methods

在vue实例中，方法通过methods配置，方法中可以通过this访问到data里的数据。

methods:{
	f1:function(){
	},
	f2(){
		this.msg=''
	},
}

上面的写法中使用了简写形式，直接用属性名(){}作用：

  • 响应视图的事件
  • 操作属性

方法操作数据的原理

  • this指向的是当前的vm实例
  • data和methods中的属性都被挂载到了vm实例上
  • 调用方法时，相当于执行了vm.方法()

4.深入理解

:::info

  1. 同名问题，data和method里的属性不能同名。
  2. 箭头函数问题，methods中尽量不使用箭头函数，箭头函数不方便对数据进行操作因为其不具有自己的this，this为父级作用域。 :::

5.总结

:::info

  • el是用来数据在什么地方展示以及操作的
  • 属性是用来保存数据的
  • 方法是用来操作数据的 :::

五、模板语法

1.什么是模板

简单来说，能被vue关联且解析的DOM中的字符串是模板。

2.模板语法的分类

分为插值语法和指令语法

3.插值语法

3.1什么是插值语法

一言以蔽之，{{}}括起来的就是插值语法。

3.2如何使用

插值语法通常使用在文本节点。{{}}内部可以写属性名，js表达式，不能写语句(定义，声明，流程语句)。

💡如何判断一个节点是文本节点

:::info 先来了解啥是节点，根据 W3C 的 HTML DOM 标准，HTML 文档中的所有内容都是节点：

  • 整个文档是一个文档节点
  • 每个 HTML 元素是元素节点
  • HTML 元素内的文本是文本节点
  • 每个 HTML 属性是属性节点
  • 注释是注释节点

上面的解释还不够清楚的话，打开vscode，调试页面打开开发者模式，在元素页面选择某一节点，可以看到$0符号，在控制台console.dir($0)，查找nodeType属性，属性值为3就是文本节点。 :::

4.指令语法

顾名思义，是基于某种指令的语法，所谓的指令就是带有v-开头的特殊属性attribute(可以包括自定义的属性)。指令语法分为一下几类。

4.1单向绑定

单向绑定绑定的是普通元素(非表单元素)的**attribute**，通常使用**v-bind**来进行绑定。其特点是V不会改变M，但操作M会改变V。

<a v-bind:href="url">某个神秘链接</a>
<a :href="simpleUrl">百度？</a>

上面的示例中用到了简写形式，即可以省去v-bind保留**: **。**attribute**的值，就是双引号里的内容，是一个指令表达式。

指令表达式是什么

:::info 指令的值就是指令表达式。所以指令表达式返回的是一个值。指令是一个特殊的属性，指令表达式就是这个属性的值。 :::

指令表达式能写什么

:::info

  1. 可以写**vm**里的属性名，从而使该指令响应式的获得属性值
  2. 由**vm**的属性名构成的表达式，注意是表达式，不能是语句
  3. 根据指令的不同，还可以写**vm**的方法名(括号问题由要不要传参决定) ::: 简单案例
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>v-bind</title>
    <script src="../node_modules/vue/dist/vue.js"></script>
</head>

<body>
    <div id="app">
        <!-- 单向绑定，操作的是普通元素，非表单元素 -->
        <a href="url">百度</a>
        <!-- 应用指令后，属性值被当作 指令表达式解析 -->
        <a v-bind:href="url">百度</a>
        <!-- 简写 省略v-bind 保留：-->
        <a :href="url">百度？</a>
        <!-- 指令表达式
            1.可以直接访问vm实例身上的属性
            2.可以是一个js表达式
            3.不能写js语句
        -->
    </div>
    <script>
        const vm = new Vue({
            el: '#app',//指定vue解析的模板
            data: {
                url: 'http://www.baidu.com',
            },

        })
    </script>
</body>

</html>

4.2 双向绑定

绑定的是表单元素，通常使用**v-model='指令表达式'**来绑定。其绑定的是表单元素的**value**属性。其特点是M,V会相互影响。双向绑定是一个语法糖，其原理就是通过**v-bind**绑定了表单的**value**属性并监听其相应的事件(文本框是input事件)。下面通过一个案例来实现双向绑定的原理。

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>双向绑定</title>
    <script src="../node_modules/vue/dist/vue.js"></script>
</head>
<body>
    <div id="app">
        <!-- 
            双向绑定 表单元素
            input type
                text
                password
                radio 单选框
                checkbox 复选框
            select 下拉选框
                option 选项
            textarea 文本域
         -->
        <!-- 
            v-mode  语法糖
            1.数据改变，会影响视图
            2.视图的改变会影响数据
         -->

        <input type="text" :value="username" v-on:input="handleInput">
    </div>
    <script>
        const vm=new Vue({
            el:'#app',
            data:{
                username:'',
            },
            methods:{
                handleInput(e){
                    // e代表输入事件的对象
                    console.log(e)
                    console.log(e.data)
                    // +=只适合只能输入，不能删除，删除会变null
                    //this.username+=e.data//输入事件是监听输入的字符，e.data只是当前输入的字符的值
                    this.username=e.target.value//e.target 获取的是dom对象，能���更准确的获得当前的输入框里的值。
                }
            }
        })
    </script>
</body>
</html>

4.5 条件渲染

分类：v-if ，v-show两者的共同点： :::info

  • 在条件成立的时候，会渲染DOM ::: 区别： :::info

  • 在条件不成立的时候，v-if不会渲染DOM，v-show会渲染DOM，但是通过display属性来隐藏DOM元素。 ::: 特点： :::info

  • v-if的空间效率更高，v-show的时间效率更高

  • v-if与v-else-if，v-else搭配适合逻辑更复杂的情况。

  • v-show适合频繁切换显示。 ::: 注意： :::info v-if与v-else-if与v-else要紧跟着。否则后面的无法被识��。就像使用if-else一样。 ::: 小示例

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>条件渲染</title>
    <script src="../node_modules/vue/dist/vue.js"></script>
</head>

<body>
    <div id="app">
        <!-- 
            条件为真没有区别都显示。
            条件为假，v-if不会渲染真实的DOM
                v-show会渲染DOM，但是会通过display:none属性隐藏。
         -->
        <!-- 
             时间和空间效率
             v-show 时间效率高，用于频繁切换显示
             v-if 空间效率高，用的时候占用空间，占空间少。适合逻辑复杂的场景。
          -->
        <!-- 
            逻辑角度
            v-if适合逻辑复杂的场景,如 v-if  v-else-if v-else,但这些需要紧跟在一起使用的,否则后面的无法被识别
         -->
        <!-- 
            v-if是真正的条件渲染，条件成立渲染DOM，不成立不渲染 
            
        -->

        <div v-if="flag">这是通过v-if渲染的</div>
        <!-- <div v-else-if="flag==true">这个是通过v-else渲染的</div>
        <div v-else>这个是通过v-else渲染的</div>

        <div v-show="flag">这是通过v-show渲染的</div>
        <div v-show="!flag">这是通过v-show渲染的</div> -->

        <button @click="toggleShow">切换显示</button>
        <button @click="flag=!flag">切换显示</button>
        <button @click="flag=flag?false:true">切换显示</button>
    </div>
    <script>
        const vm = new Vue({
            el: '#app',
            data: {
                flag: true,
            },
            methods: {
                toggleShow() {
                    this.flag = !this.flag
                }
            }
        })
    </script>
</body>

</html>

4.6 列表渲染

1) 什么是列表渲染

我们可以用** v-for** 指令基于一个可迭代对象来渲染一个列表(li，等dom元素)。

2）语法

v-for='循环变量 in 迭代对象'

3）说明

:::info

  • 当迭代对象为数组的时候，循环变量可以有两个参数，第一个为当前遍历到的元素，第二个为当前遍历到的元素的索引，只有一个不需要加括号。 - **v-for='(item,index) in arr'** - **v-for='item in arr'**
  • 当迭代对象为对象的时候，遍历的为对象的属性(方法也是对象的一种属性也会被遍历)。循环遍历第一个参数为属性值，第二个参数为属性名，还可以有第三个参数为当前遍历到的属性的下标。 - **v-for='(value,key,index) in obj'** - **v-for='(value,key) in obj'** - **v-for='value in obj'** ::: 示例：
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>列表渲染</title>
    <script src="../node_modules/vue/dist/vue.js"></script>
</head>

<body>
    <div id="app">
        <!-- 语法 循环变量 in 可迭代对象 -->
        <ul>
            <!-- 只获取数组的元素 -->
            <li v-for="item in items">{{item}}</li>
        </ul>

        <!-- 获取数组下标 -->
        <div v-for="(item,index) in items">{{index}}--{{item}}</div>

        <!-- 
            遍历对象
                第一个参数是属性值
                第二个参数是属性名
                第三个参数是下标 基本不用
         -->
        <div v-for="value in obj">{{value}}</div>
        <div v-for="(value,key) in obj">{{key}}--{{value}}</div>
    </div>
    <script>
        const vm = new Vue({
            el: '#app',
            data: {
                items: ['item1', 'item2', 'item3'],
                obj: {
                    attr1: '111',
                    attr2: '222',
                    attr3: '333',
                    fn: function () { },
                },
                users: [
                    { id: 1, name: '小米' },
                    { id: 2, name: '小胖' },
                    { id: 3, name: '小强' },
                ]
            },
            methods: {
                add() {
                    this.users.unshift({ id: 4, name: '小明' })
                }
            }
        })
    </script>
</body>

</html>

六、计算属性computed与侦听器watch

1.介绍

:::info

  • 计算属性是一个属性，会挂载到vm实例上。

  • 作用：对原始数据进行再次加工后输出。

  • 特点：计算属性存在缓存，方便调试，是响应式的。 ::: :::info

  • 侦听器是一个用于观察某一数据变动的属性，其并不会挂载到vm实例上。

  • 在侦听的数据发生变化时执行相应的回调函数 :::

2.语法

计算属性：**computed:{key:function|key:Object|key(){}}**示例

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>练习 计算属性</title>
    <script src="../node_modules/vue/dist/vue.js"></script>
</head>

<body>
    <div id="app">
        姓：<input type="text" v-model="lastname">
        <br>
        名：<input type="text" v-model="firstname">
        <br>
        全名:{{name}}
        <br>
        全名:{{name2}}
        <br>
        全名:{{name3}}

    </div>
    <script>
        const vm = new Vue({
            el: '#app',
            data: {
                lastname: '',
                firstname: '',
            },
            computed: {
                name: {
                    get: function () {
                        return this.lastname + this.firstname
                    }
                },
                name2: function () {
                    return this.lastname + this.firstname
                },
                name3() {
                    return this.lastname + this.firstname
                }
            }
        })
        console.log(vm);
    </script>
</body>

</html>

侦听器：**watch:{Function|key:Object}**

<!DOCTYPE html>
<html lang="en">

            //监听器可以获得变化的情况，通过给函数穿参，监听器(newValue,oldValue)
            // watch 可以开启深度监听
            watch: {

                },

                    // deep: true,
                    handler: function () {
                        console.log(this)
                        console.log('stu is changed...');
                    }

                }
            },
            computed: {
                name: {
                    get: function () {
                        return this.lastname + this.firstname

                    return this.lastname + this.firstname

3.过滤器

是一个函数，完成数据的格式化。有全局过滤器和器两种语法： :::info 全局：**Vue.filter(String,Function)**第一个参数为过滤器名，第二个为过滤方法局部：**filters:{Function,...}** ::: 可以进行链式操作，即多次格式化同一个数据，这是相对于computed的差距。

            return msg + '元'

        const vm = new Vue({

七、插槽

插槽 slot

什么是插槽，供可替换性的组件使用，可以提供程序的扩展性使用步骤： :::info

  1. 预留插槽

  2. 替换插槽 ::: 具名插槽使用步骤 :::info

  3. 预留插槽，起名 **name=""**

  4. 替换插槽，确定插槽类型 slot=“name” ::: 插槽匹配机制 :::info

  5. 首先先判断要插入插槽的内容的插槽名称**slot="name"**，没有类型默认为default

  6. 寻找是组件上否有预留这样类型的插槽

  7. 有就替换相应的位置

  8. 没有就放入默认插槽，**name="default"**，或者没有指定name的插槽

  9. 有多个同名插槽，则匹配的内容就会将所有都替换一遍 :::

<div class="city-item-container">
  <slot></slot>
  <slot name='city' />
</div>

使用组件，默认插槽

<CityItem>
  写在这里的内容都会替换<slot></slot>
</CityItem>

<CityItem>
  <div slot="city">hello</div>
</CityItem>

注意，插槽的作用域是可以访问到父组件的作用域的，即父组件中定义的数据可以使用在子组件中。
