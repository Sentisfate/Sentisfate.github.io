vue3 使用挂载路由的方法，注意对应的是vue-router4

// 导入相关模块
import * as vueRouter from 'vue-router';
import HomeCom from '../views/HomeCom.vue';
import WelcomCom from '../views/WelcomCom.vue';
// 注册vuerouter插件

// 定义路由规则
const routes = [
  {

    path: '/home',

  },

// 创建路由对象
const router = new vueRouter.createRouter({

export default router;

const app = createApp(App);
app.use(router);
app.mount('#app');

3.使用vue3

  1. 安装 **pnpm i vue-router**
  2. 定义路由组件
  3. 定义路由规则**routes**
  4. 创建路由实例**router**并传递**routes**配置 **createRouter()**
  5. 挂载根实例，注意挂载到app上
// 1. 定义路由组件.
// 也可以从其他文件导入
const Home = { template: '<div>Home</div>' }
const About = { template: '<div>About</div>' }

// 2. 定义一些路由
// 每个路由都需要映射到一个组件。
// 我们后面再讨论嵌套路由。
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
]

// 3. 创建路由实例并传递 `routes` 配置
// 你可以在这里输入更多的配置，但我们在这里
// 暂时保持简单
const router = VueRouter.createRouter({
  // 4. 内部提供了 history 模式的实现。为了简单起见，我们在这里使用 hash 模式。
  history: VueRouter.createWebHashHistory(),
  routes, // `routes: routes` 的缩写
})


// 5. 创建并挂载根实例
const app = Vue.createApp({})
//确保 _use_ 路由实例使
//整个应用支持路由。
app.use(router)
app.mount('#app')

// 现在，应用已经启动了！

:::info 如果直接按照上面的组件写法，const Home = { template: '<div>Home</div>' }，可能会报下面的错误，导致路由无法使用。Component provided template option but runtime compilation is not supported in this build of Vue. Configure your bundler to alias “vue” to “vue/dist/vue.esm-bundler.js”.

由于我是vite构建的项目，解决方案是在vite.config.js里配置如下： :::

import { defineConfig } from "vite";
import vue from "@vitejs/plugin-vue";

// https://vitejs.dev/config/
// export default defineConfig({
//   plugins: [vue()],
// });

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      'vue': 'vue/dist/vue.esm-bundler.js'
    }
  }
})

在setup中访问路由

使用**useRouter**来代替**this.$router**,**useRoute**代替**this.$route**

import { useRouter, useRoute } from 'vue-router'

export default {
  setup() {
    const router = useRouter()
    const route = useRoute()

    function pushWithQuery(query) {
      router.push({
        name: 'search',
        query: {
          ...route.query,
        },
      })
    }
  },
}

路由守卫工作原理

其有什么作用呢？打个比方(个人理解)：

甲要去A城，去A城要通行证，甲没有通行证，到A城前被大数据查到，被禁止进入A城，大数据要求甲到A城里B地办理通行证。于是甲若想到A城只能到B地办理通行证。 一个网站，里面有众多资源，其中有非公开(并非所有人都可以访问)，和公开的资源，url可以定义任一个资源，我们想要非公开资源不被随意访问，除非具有某一”通行证“，路由守卫的作用就是保护非公开资源。其判断条件常有两种，

  1. 访问的是否是公开资源
  2. 是否具有通行证
// 设置全局前置导航守位
router.beforeEach((to, from, next) => {
  // to 需要访问的path,to.path 是只读的
  // from 从哪里来
  // next() 放行  前往to指定的页面
  // localStorage---本地存储，是一个对象，可以理解成字典，存储的是键值对密钥
  let isLogin = localStorage.getItem('isLogin') == 1 ? true : false;
  console.log(isLogin);
  console.log(to.path);
  console.log(from.path); //上一个页面的路径，刷新的话为/
  //判断访问的是否是公开资源，这里是是否是登录页面
  if (to.path == '/login') {
    //是的话放行
    next();
    return;
  }
  //非公开资源时，再判断是否登录，也就是有通行证
  if (isLogin) {
    //是的话放行
    next();
  } else {
    //否则的话，前往登录页面
    next('/login');
  }
});

路由的存储原理

路由存储在栈的结构A页面跳转到B页面，destroy生命周期和create周期，哪个在前，哪个在后。

同步路由异步路由

同步路由的写法:

import AboutView from '@/views/AboutView';

...
  {
    path: '/about',
    name: 'about',
    component: AboutView,
  },

异步路由的写法:

  {
    path: '/city',
    name: 'city',
    component: () => import('@/views/CityView.vue'),
  },

区别:是按需导入

  • 同步路由会一次性将路由对应的组件集中同步加载。
  • 异步路由，加载到对应的路由模块，才会开始加载。也叫做懒加载。

异步路由的好处，减少首屏加载的时间

keep-alive

让组件保持会话状态，不销毁

  1. keep-alive包裹的router-view，组件的生命周期只会触发一次，不会再因路由跳转而销毁，但是可以手动销毁this.$destroy
  2. keep-alive 将不活动的组件实例缓存起来，mounted，destroy等生命周期不会再重复触发

  1. keep-alive 包裹的router-view 对应的组件会多两个生命周期 activated deactivated ，执行顺序是 一个组件活动状态->不活动状态->另一个组件活动状态
  2. 不想要某个组件缓存，给组件加一个**name**属性，并且在**keep-alive**上加一个**exclude**参数
//给组件设置name属性
  name: 'DetailView',

<keep-alive exclude="DetailView">
    <router-view />
</keep-alive>
  1. 一个页面滚动，对另一个页面造成影响，scrollBehavior，函数返回一个对象，有x，y属性，表示滚动条的位置，跳转的时候，滚动条会在定位的位置
 scrollBehavior(to, from, savedPosition) {
    console.log('to:');
    console.log(to, from);
    if (to.path == '/detail') {
      return { x: 0, y: 0 };
    }
    console.log('savedPosition');
    // savedPostion 只会在浏览器导航栏前进后退时触发
    console.log(savedPosition);
    return savedPosition;
  },
