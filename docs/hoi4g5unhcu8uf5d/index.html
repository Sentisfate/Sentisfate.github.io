纹理

2023-07-06 02:11

webgl中纹理通常是二维矩阵数据，可以传递给着色器使用，可以像下面这样声明一个均匀采样器。

uniform sampler2D someTexture;

纹理坐标系 ST

纹理也有自己的坐标范围，无论怎样的尺寸大小，纹理的空间范围是0-1，左下角为原点，水平向右为s轴正方向，垂直向上为t轴正向，右上角点为（1，1）

对于超出纹理空间的坐标，webgl默认是平铺处理，当然也可以用gl.texParameteri修改纹理参数来改变处理方式。

纹理坐标系是理想状态，直接渲染（图片像素数据源）纹理时 webgl 使用的图片坐标系。因此在向缓冲区对象中传递几何图形的点位和纹理图像的点位时，应该使用图片坐标系作为参考。图片坐标系与屏幕坐标系一致。但在使用帧缓冲中的纹理时，坐标系为纹理坐标系。

纹理单元

纹理单元用于确定采样器具体使用哪个纹理，可以看成是纹理的引用，webgl中有多个纹理单元，可以把它们看成是引用纹理的全局数组。

const gl = {
  activeTextureUnit: 0,
  textureUnits: [
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
    { TEXTURE_2D: null, TEXTURE_CUBE_MAP: null, },
  ];
}

如上所示，默认情况下，纹理单元是没有绑定纹理对象的，因此在使用前，需要绑定纹理。

// 创建纹理对象
const tex = gl.createTexture();

const unit = 1; //挑选一个纹理单元
// 激活某个纹理单元
gl.activeTexture(gl.TEXTURE0 + unit);//gl里默认活动的纹理单元是gl.TEXTURE0，不显式激活时默认会被绑定到纹理单元TEXTURE0上
// 将纹理单元与纹理对象绑定
gl.bindTexture(gl.TEXTURE_2D, tex);

绑定后需要进行纹理数据的写入，将内存中的纹理数据通过绑定点gl.TEXTURE_2D写入到GPU中绑定的纹理对象上。

gl.texImage2D(
  gl.TEXTURE_2D,
  0,
  gl.RGBA,
  gl.RGBA,
  gl.UNSIGNED_BYTE,
  image,//图像可以作为纹理的像素源
);

最后需要告诉gl纹理采样器对应哪个纹理单元。

// 将纹理采样器与纹理单元相绑定,不绑定默认与纹理单元0绑定
gl.uniform1i(imgLocation, unit);

纹理单元的个数是固定的，由GPU决定，不同GPU可以具备不同的硬件架构和资源分配策略，对纹理单元的数量有着不同的限制，webgl要求至少支持8个纹理单元。可通过下面的方式获取相关参数。

const maxTextureUnits = gl.getParameter(gl.MAX_COMBINED_TEXTURE_IMAGE_UNITS);

顶点着色器和片段着色器有着不同的数量限制：

const maxVertexShaderTextureUnits = gl.getParameter(gl.MAX_VERTEX_TEXTURE_IMAGE_UNITS);
const maxFragmentShaderTextureUnits = gl.getParameter(gl.MAX_TEXTURE_IMAGE_UNITS);

但是其总和不超过总个数。此外有的设备可能不支持在顶点着色器中使用纹理，最好在使用前查一下。

关于纹理相关函数的一些伪代码实现

引用自https://webglfundamentals.org/webgl/lessons/zh_cn/webgl-texture-units.html

// 伪代码！！！
gl.activeTexture = function(unit) {
  gl.activeTextureUnit = unit - gl.TEXTURE0;  // 转成从 0 开始的索引
};
 
gl.bindTexture = function(target, texture) {
  const textureUnit = gl.textureUnits[gl.activeTextureUnit];
  textureUnit[target] = texture;
}:

// 伪代码！！！
gl.texImage2D = function(target, level, internalFormat, width, height, border, format, type, data) {
  const textureUnit = gl.textureUnits[gl.activeTextureUnit];
  const texture = textureUnit[target];
  texture.mips[level] = convertDataToInternalFormat(internalFormat, width, height, format, type, data);
}
 
gl.texParameteri = function(target, pname, value) {
  const textureUnit = gl.textureUnits[gl.activeTextureUnit];
  const texture = textureUnit[target];
  texture[pname] = value; 
}
