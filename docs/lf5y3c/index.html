==与===

==会触发隐式转换

对象转数字

  1. 先尝试调用对象的valueOf方法，如果有就返回返回值
  2. 调用对象的toString方法，返回将返回值原始值转换为数字
  3. 否则就抛出类型错误

对象转字符串

  1. 先尝试调用对象的toString方法，将返回值转为字符串
  2. 无toString或者toString返回值非原始值，就调用valueOf，将返回值转为字符串
  3. 若valueOf返回值非原始值，就抛出类型错误

js的原始��为基本数据类型，即为number，string，boolean，null，undefined这五种，指的是存放在栈中的数据

对象转boolean 为true

数组

隐式转换会默认调用join的方法，每个元素之间用，分割

console.log() 打印数组时用的是toString，打印字符串用的是valueOf

null==undefined

为true，不会发生隐式转换，===为false

常见类型转换

infinity 表示正无穷

注意易错点：对象转字符串，null，undefined转数字，数组转 [] 字符串，转数字

get和Object.defineProperty对比

相同点：

  • 都可以定义属性被查询时的函数

不同点在class内部使用时

  • get属性会被定义到实例的原型上
  • Object.defineProperty将定义在实例自身上

对比escape encodeURI 和encodeURIComponent

  1. escape(已废弃) 对字符串编码，ascii字母，数字，@ * / + - _ .之外的字符都被编码
  2. encodeURI 编码URLascii字母，数字，@ * / + 和 ~ ! # $ & () =, ; ?- _ . '之外的字符都被编码
  3. encodeURIComponent，ascii字母，数字，~ ! * ( ) - _ . ’ 之外的字符被编码
