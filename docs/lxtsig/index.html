事件

2022-09-12 08:48

事件冒泡

  • DOM0和IE支持
  • 从事件源向父级，一直到根元素
  • 某元素某类型事件被触发，父元素同类型事件同样会触发

事件捕获

  • DOM2支持
  • 从根元素HTML到事件源
  • 某个元素的的某类型事件触发，先触发根元素，再一级一级出发到子元素

事件流

事件的流向：事件捕获，事件源，事件冒泡

阻止事件冒泡

  • 标准：event.stopPropagation()
  • IE:event.cancelBubble=true

Event Loop的执行顺序

宏任务

  • Task Queue
  • setTimeOut,setInterval,setImmediate,I/O(ajax操作),script,UI rendering

微任务

  • Job Queue
  • 浏览器Promise,promise.then,MutationObserver,Promise.resolve().then,
  • Node.js：process.NextTick

异步任务中的优先级：process.nextTick>Promise>promise.then>setTimeout>setImmediate

执行顺序

  • 首先执行同步代码
  • 执行微任务队列，清空后
  • 检查是否需要进行浏览器的更新
  • 重新开始事件循环，执行一个宏任务

为什么vue.$nextTick的优先级比setTimeout高

vue.$nextTick优先使用的异步函数是promise,MutationObserve,setImmediate，优先级高
