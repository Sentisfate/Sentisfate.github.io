语雀文档同步到hexo

2023-07-16 21:36

tags: [‘hexo’,‘语雀’]categories: ‘杂项’cover: ‘https://cdn.jsdelivr.net/gh/Sentisfate/wanhuatong@main/img/loading.gif'date: 2022-11-14

平常的话，一般都是在用语雀记笔记总结之类的，说实话，个人感觉写作体验非常好。这时候就在想，能不能将语雀文档转移到 hexo 博客上呢。一开始自己尝试的是手动转移，试了一篇，太累了不说，图片还不能显示。能否实现一键转移呢，网上随便搜索了一下，就发现了它yuque-hexo。下面介绍下具体的实现过程。(默认大家都用过语雀了哈哈)github 官方教程地址:GitHub - x-cold/yuque-hexo: 同步语雀的文章到你的 Hexo 项目吧！

  1. npm i yuque-hexo 安装到博客根目录
  2. 安装完后，在 package.json，添加自己语雀信息相关的配置，主要是**yuqueConfig****那一块。(**其余有些配置也是做了一些优化)

参数名 含义 默认值
postPath 文档同步后生成的路径 source/_posts/yuque
cachePath 文档下载缓存文件 yuque.json
lastGeneratePath 上一次同步结束的时间戳的文件
mdNameFormat 文件名命名方式 (title / slug) title
adapter 文档生成格式 (hexo/markdown) hexo
concurrency 下载文章并发数 5
baseUrl 语雀 API 地址 -
login 语雀 login (group), 也称为个人路径 -
repo 语雀仓库短名称，也称为语雀知识库路径 -
onlyPublished 只展示已经发布的文章 false
onlyPublic 只展示公开文章 false
imgCdn 语雀图片转 CDN 配置

其中 **login**和**repo**字段，参照 知识库设置里的信息

  1. 生成图床 token ，本人用的是 github 图床
  2. 生成语雀 token，授权范围此处我是多勾了，应该只需要图集，文档和知识库这三个权限即可。

  1. 在博客根目录新建环境变量文件.env 文件，记得在.gitnore 文件中配置忽略.env，并配置如下：
YUQUE_TOKEN=xxx   --语雀token
SECRET_ID=xxx   --github 用户名
SECRET_KEY=xxx  --github图床token
  1. 安装 dotenv-cli(将环境变量从.env 加载到 process.env)，npm i dotenv-cli 就可以通过dotenv -e .env将环境变量传递给程序了。为了方便，我们在 package.json 里配置 scripts，这部分不了解的可以看下这里

https://www.ruanyifeng.com/blog/2016/10/npm_scripts.html

  "scripts": {
    "build": "hexo generate",
    "sync": "dotenv -e .env yuque-hexo sync",
    "cly": "yuque-hexo clean",
    "clean": "hexo clean",
    "deploy": "hexo clean && hexo generate && hexo deploy",
    "serve": "hexo clean && hexo generate && hexo server"
  },

说明：

  1. yuque-hexo sync会将语雀指定的知识库里的文章同步到指定的目录里。
  2. yuque-hexo clean 会清除缓存，即生成的yuque.json及其对应的文件

到这里基本上就配置完毕了，可以使用npm run sync来测试能否同步了。

顺带一提，front-matter 的配置也可以在语雀中这样写，即用分割线分隔 front-matter 和正文，注意要设置转换格式为**hexo**。
